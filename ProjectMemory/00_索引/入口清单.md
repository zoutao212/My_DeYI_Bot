# 入口清单（最小集：Control UI / Config）

> 目标：从“页面/按钮/现象”快速定位到**唯一入口文件**与**关键函数/方法**。

## Web 控制台（Control UI）页面入口

### `/config`（配置页）

- 页面渲染入口：`ui/src/ui/app-render.ts` -> `renderConfig({...})`
- 页面组件：`ui/src/ui/views/config.ts` -> `renderConfig(props)`
- 数据加载（RPC）：`ui/src/ui/controllers/config.ts`
  - `loadConfig(state)` -> `client.request("config.get", {})`
  - `loadConfigSchema(state)` -> `client.request("config.schema", {})`
  - `saveConfig(state)` -> `client.request("config.set", { raw, baseHash })`
  - `applyConfig(state)` -> `client.request("config.apply", { raw, baseHash, sessionKey })`

### `/chat`（聊天页）

- 页面渲染入口：`ui/src/ui/app-render.ts` -> `renderChat({...})`
- 顶部控制条（含“思考输出”切换按钮）：`ui/src/ui/app-render.helpers.ts` -> `renderChatControls(state)`
  - 关键开关：`state.settings.chatShowThinking`（LocalStorage 持久化）

## Gateway RPC（后端处理入口）

### Config 相关 RPC

- RPC 注册入口：`src/gateway/server-methods/config.ts` -> `configHandlers`
  - `"config.get"`：读取配置快照（含解析+校验+hash）
  - `"config.schema"`：生成配置 JSON Schema + UI hints
  - `"config.set"`：写入配置文件
  - `"config.apply"`：应用配置（可能触发 reload / restart 策略）

### Config 文件 IO（真实读写入口）

- 读快照（UI 展示用）：`src/config/io.ts` -> `readConfigFileSnapshot()`
- 读配置（运行态加载）：`src/config/io.ts` -> `loadConfig()`
- 写配置（落盘）：`src/config/io.ts` -> `writeConfigFile(cfg)`

## 本次涉及的配置字段（UI/config 相关）

- `models.providers.*`：第三方 OpenAI 兼容 Provider（例如 `vectorengine`）
- `agents.defaults.model.primary`：默认主模型
- `agents.defaults.models`：默认模型字典
- `agents.defaults.memorySearch.*`：记忆检索/嵌入配置（包含 remote baseUrl/model/apiKey 等）

## Agent 大脑（LLM 请求调度）入口

### Agent 运行主入口

- **核心入口**：`src/agents/pi-embedded-runner/run.ts` -> `runEmbeddedPiAgent(params)`
  - 职责：管理整个 Agent 运行生命周期
  - 参数：`RunEmbeddedPiAgentParams`（包含 sessionId, prompt, model, config 等）
  - 返回：`EmbeddedPiRunResult`（包含 payloads, meta, usage 等）

- **执行尝试**：`src/agents/pi-embedded-runner/run/attempt.ts` -> `runEmbeddedAttempt(params)`
  - 职责：执行单次 LLM 请求尝试
  - 关键步骤：
    1. 加载 Session Manager
    2. 清理和限制历史消息
    3. 生成会话摘要
    4. 构建 System Prompt
    5. 调用 LLM API
    6. 处理响应并保存

### System Prompt 构建

- **主函数**：`src/agents/system-prompt.ts` -> `buildAgentSystemPrompt(params)`
  - 职责：构建 Agent 的"指令集"
  - 包含：身份、工具、技能、文档、运行时信息、会话摘要
  - 动态生成：根据 promptMode（full/minimal/none）、provider、channel 等

### 对话历史管理

- **历史限制**：`src/agents/pi-embedded-runner/history.ts` -> `limitHistoryTurns(messages, limit)`
  - 职责：保留最近 N 轮对话，减少 token 消耗
  - 关键改进：保留第一条 user 消息（任务目标）

- **DM 历史限制配置**：`getDmHistoryLimitFromSessionKey(sessionKey, config)`
  - 职责：从配置中读取 DM 会话的历史限制
  - 配置路径：`config.channels.<provider>.dmHistoryLimit`

### 会话摘要机制

- **生成摘要**：`src/agents/session-summary.ts` -> `generateSessionSummary(messages)`
  - 职责：从历史对话中提取任务上下文
  - 提取内容：任务目标、已执行操作、关键决策、遇到的问题

- **格式化摘要**：`formatSessionSummary(summary)`
  - 职责：将摘要格式化为 Markdown 文本
  - 注入位置：System Prompt 的 Runtime 部分之前

### Session Manager（持久化）

- **核心类**：`SessionManager` (Pi Agent Core)
  - 职责：保存和加载所有历史消息
  - 文件格式：`~/.clawdbot/sessions/<sessionId>.jsonl`
  - 关键方法：
    - `SessionManager.open(sessionFile)`：打开 session 文件
    - `buildSessionContext()`：返回消息列表
    - ⚠️ **已知问题**：可能返回空数组（缓存问题）

### 数据流快速参考

```
用户输入
  ↓
runEmbeddedPiAgent (run.ts)
  ↓
runEmbeddedAttempt (attempt.ts)
  ↓
SessionManager.open() → buildSessionContext()
  ↓
limitHistoryTurns() → 保留最近 N 轮 + 任务目标
  ↓
generateSessionSummary() → 提取任务上下文
  ↓
buildAgentSystemPrompt() → 注入会话摘要
  ↓
createAgentSession() → 初始化 Pi Agent Core
  ↓
LLM API 调用
  ↓
处理响应 → 保存到 SessionManager
  ↓
返回给用户
```
