# 🤖 自动化配置说明

本文档记录了 Project Memory Gardening 和 Maintain Rules 两个 Power 的自动化配置。

---

## 📋 已实施的自动化方案

### ✅ 方案 1：Steering Rules 集成（持久化）

**位置：** `.kiro/steering/always/always_workflow.md`

**效果：** AI 助手在所有对话中都会遵循这些自动触发规则

#### 自动触发场景

**代码地图维护（Project Memory Gardening）：**
- 新增/变更入口（API、启动脚本、任务入口）
- 新增/重构模块（目录结构、职责边界）
- 修复复杂 Bug（需要记录调用链）
- 用户明确要求（"不要扫全库"、"快速定位"）

**规则与技能固化（Maintain Rules）：**
- 重复模式出现（≥3 次）
- 发现"事实与代码不一致"
- 规则/流程缺口导致错误
- 用户明确要求（"记住这个套路"、"下次直接用"）

#### 任务完成后的自动回顾

每次完成任务后，AI 会自动检查：
1. 是否需要更新代码地图？
2. 是否出现了可复用模式？
3. 是否需要补充规则？

---

### ✅ 方案 2：Agent Hooks（事件驱动）

**位置：** `.kiro/hooks/` 目录

#### Hook 1: 自动更新代码地图

**ID：** `auto-update-code-map`  
**触发事件：** 文件编辑（fileEdited）  
**监控文件：** `src/**/*.ts`, `src/**/*.js`, `server/**/*.py`, `apps/**/*.swift`, `apps/**/*.kt`

**功能：**
- 代码修改后自动检查是否需要更新代码地图
- 检查是否新增/修改了入口
- 检查是否新增/重构了模块
- 检查是否修改了关键调用链

**行动：**
- 激活 project-memory-gardening Power
- 更新 `ProjectMemory/00_索引/` 下的相关文件

#### Hook 2: 自动捕获可复用模式

**ID：** `auto-capture-patterns`  
**触发事件：** 任务完成（agentStop）

**功能：**
- 任务完成后自动回顾对话
- 识别重复的定位流程（≥3 次）
- 识别固定的调试步骤
- 识别"事实与代码不一致"的情况
- 识别可以固化的套路或最佳实践

**行动：**
- 激活 maintain-rules Power
- 提炼模式（问题、原因、解法）
- 固化到对应文件（规则/代码地图/技能）

#### Hook 3: 自动记录新文件

**ID：** `auto-record-new-files`  
**触发事件：** 文件创建（fileCreated）  
**监控文件：** `src/**/*.ts`, `src/**/*.js`, `server/**/*.py`, `apps/**/*.swift`, `apps/**/*.kt`, `*.sh`, `*.bat`, `*.ps1`

**功能：**
- 新建文件后自动检查是否需要记录到代码地图
- 检查是否是新的入口文件
- 检查是否是新的模块文件
- 检查是否是配置文件

**行动：**
- 激活 project-memory-gardening Power
- 更新入口清单/模块地图/关键词索引

---

### ✅ 方案 3：关键词激活（内置）

**位置：** Power 的 frontmatter 关键词配置

**Project Memory Gardening 关键词：**
- 代码地图、项目记忆、快速定位、入口清单、调用链、索引
- code map, project memory

**Maintain Rules 关键词：**
- 规则维护、技能优化、自我成长、模式提炼、认知构建
- rules, skills, self-improvement

**效果：** 当你的消息中包含这些关键词时，AI 会主动激活相应的 Power

---

## 🎯 自动化层级

```
层级 1: 关键词激活（最轻量）
  ↓
  用户消息包含关键词 → AI 主动激活 Power
  
层级 2: Steering Rules（持久化）
  ↓
  AI 在所有对话中遵循自动触发规则
  
层级 3: Agent Hooks（事件驱动）
  ↓
  文件编辑/创建/任务完成 → 自动触发检查
```

## 🔧 管理与维护

### 查看 Hooks 状态

在 Kiro IDE 中：
1. 打开命令面板（Ctrl+Shift+P）
2. 搜索 "Kiro Hook"
3. 选择 "Open Kiro Hook UI"

或者在 Explorer 视图中查看 "Agent Hooks" 部分。

### 启用/禁用 Hooks

在 Hook UI 中可以：
- 查看所有 Hooks
- 启用/禁用特定 Hook
- 编辑 Hook 配置
- 删除 Hook

### 查看 Steering Rules

**位置：** `.kiro/steering/always/always_workflow.md`

可以随时编辑此文件来调整自动触发规则。

## 🚀 使用建议

### 初期（前 1-2 周）

- 让所有自动化都保持启用
- 观察 AI 的自动触发行为
- 根据实际效果调整规则

### 稳定期（2 周后）

- 代码地图应该已经比较完善
- 常见模式应该已经固化
- 可以考虑禁用某些不常用的 Hook

### 长期维护

- 定期回顾代码地图的完整性
- 定期回顾规则的有效性
- 删除过时的规则和技能

---

## 📝 反馈与改进

如果发现自动化有问题：

1. **太频繁触发**：调整 Hook 的文件匹配模式
2. **触发不及时**：检查 Steering Rules 是否生效
3. **效果不明显**：检查代码地图是否真的在更新

可以随时调整配置，找到最适合你的自动化程度。
