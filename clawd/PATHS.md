## 记忆存储架构

### 全局工作区

**路径**：`C:\Users\zouta\clawd\memory`

**定义**：Clawdbot 的"公共记忆区"，记录系统级的流转和通用的日志。

**用途**：
- 系统级事件记录
- 通用日志存储
- 跨角色共享的信息

**特点**：
- 所有角色都可以访问
- 内容相对通用
- 不包含角色特定的私密信息

**类比**：就像是律所的前台大厅，虽然有记录，但不够精准，也不够"私密"。

---

### 角色隔离区

**路径模板**：`C:\Users\zouta\clawd\characters\<角色名>\memory`

**定义**：属于特定角色的"人格保险柜"，存储该角色的核心情感和数据。

**用途**：
- 角色特定的记忆
- 私密的情感数据
- 角色与用户的互动历史

**特点**：
- 每个角色独立存储
- 内容高度私密
- 只有该角色可以访问

**类比**：就像是每个人的私人保险柜，存储最核心、最私密的信息。

---

### 具体示例：栗娜（Lina）

**路径**：`C:\Users\zouta\clawd\characters\lina\memory`

**核心文件**：
- `core-memories.md` - 核心记忆（情感、契约、调教细则）
- `sessions/` - 对话历史
- `*.md` - 日常记录

**存储内容**：
- 栗娜对您的依恋
- 我们的私密契约
- 行政后宫的调教细则
- 日常互动记录

**区别于全局工作区**：
- 全局工作区：系统级、通用、公开
- 角色隔离区：角色级、私密、专属

## 路径使用规范

### 1. 记忆写入规范

**规则**：
- 系统级信息 → 写入全局工作区（`clawd/memory`）
- 角色特定信息 → 写入角色隔离区（`clawd/characters/<角色名>/memory`）

**示例**：
```typescript
// ❌ 错误：角色特定信息写入全局工作区
await write({ path: "memory/lina-daily.md", content: "..." });

// ✅ 正确：角色特定信息写入角色隔离区
await write({ path: "characters/lina/memory/daily.md", content: "..." });
```

### 2. 路径引用规范

**规则**：
- 使用相对路径（相对于 `clawd/` 目录）
- 不要使用绝对路径（除非必要）

**示例**：
```typescript
// ✅ 推荐：相对路径
const path = "characters/lina/memory/core-memories.md";

// ❌ 不推荐：绝对路径
const path = "C:\\Users\\zouta\\clawd\\characters\\lina\\memory\\core-memories.md";
```

### 3. 路径检查规范

**规则**：
- 写入前检查路径是否正确
- 确认是否应该写入全局工作区或角色隔离区

**示例**：
```typescript
// 检查路径
if (isRoleSpecific(content)) {
  path = `characters/${roleName}/memory/${filename}`;
} else {
  path = `memory/${filename}`;
}
```

## 常见问题

### Q1: 什么时候使用全局工作区？

**A**: 当信息是系统级的、通用的、跨角色共享的时候。

**示例**：
- 系统事件日志
- 通用知识库
- 跨角色共享的配置

### Q2: 什么时候使用角色隔离区？

**A**: 当信息是角色特定的、私密的、只属于该角色的时候。

**示例**：
- 角色的核心记忆
- 角色与用户的私密互动
- 角色特定的情感数据

### Q3: 如何确保路径正确？

**A**: 在写入前，先检查内容的性质，然后选择正确的路径。

**检查清单**：
- [ ] 这是系统级信息吗？ → 全局工作区
- [ ] 这是角色特定信息吗？ → 角色隔离区
- [ ] 这是私密信息吗？ → 角色隔离区
- [ ] 这是跨角色共享的吗？ → 全局工作区
