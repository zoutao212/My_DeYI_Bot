# Lessons Learned

项目开发过程中积累的经验教训，按主题分类，支持关键词搜索和按需加载。

## 核心价值

- 📚 **避免重复踩坑**：记录已解决的问题和解决方案
- 🔍 **快速定位**：通过关键词快速找到相关经验
- 💡 **按需加载**：只加载需要的部分，节省 token
- 🎯 **持续积累**：随着项目发展不断补充新经验

## 使用场景

### 场景 1：遇到工具调用问题

**问题**：`strReplace` 显示成功，但文件没有实际修改

**查找路径**：
1. 查看 `README.md` 索引
2. 找到 `07_AI工具使用陷阱.md`
3. 阅读"工具调用成功 ≠ 文件实际写入"章节

### 场景 2：构建后代码不生效

**问题**：修改了 TypeScript 源码，但运行时没有变化

**查找路径**：
1. 查看 `README.md` 索引
2. 找到 `10_构建验证流程.md`
3. 按照标准流程验证

### 场景 3：配置不生效

**问题**：添加了配置项，但系统没有使用

**查找路径**：
1. 查看 `README.md` 索引
2. 找到 `12_配置项验证方法论.md`
3. 按照三步法验证

### 场景 4：创建 Power 失败

**问题**：创建的 Power 无法安装

**查找路径**：
1. 查看 `README.md` 索引
2. 找到 `13_Kiro_Power创建规范.md`
3. 检查文件限制和 frontmatter 格式

## 使用方法

### 方式 1：通过索引查找（推荐）

1. 打开 `.kiro/lessons-learned/README.md`
2. 根据问题类型或关键词找到对应文件
3. 只读取需要的文件

### 方式 2：关键词搜索

使用 `grepSearch` 在 `.kiro/lessons-learned/` 目录中搜索关键词：

```bash
grepSearch -query "工具调用" -includePattern ".kiro/lessons-learned/**/*.md"
```

### 方式 3：AI 助手自动查找

AI 助手会根据问题类型自动查找相关文件：

- 遇到工具调用问题 → 自动读取 `07_AI工具使用陷阱.md`
- 遇到构建问题 → 自动读取 `10_构建验证流程.md`
- 遇到配置问题 → 自动读取 `12_配置项验证方法论.md`

## 添加新经验

当遇到新问题并解决后，按照以下步骤添加：

1. **确定主题**：这个经验属于哪个主题？
2. **选择文件**：如果是现有主题，添加到对应文件；如果是新主题，创建新文件
3. **记录内容**：
   - 问题是什么
   - 为什么会发生
   - 如何解决
   - 如何避免
4. **更新索引**：在 `README.md` 中添加关键词和快速查找条目
5. **更新版本**：增加版本号和变更说明
