1. 核心问题
vectorengine 后台错误：

Function call is missing a thought_signature in functionCall parts.
关键词：functionCall（Gemini 格式），不是 tool_calls（OpenAI 格式）

2. 根本原因
vectorengine 是一个混合 API：

Endpoint：OpenAI Completions (/v1/chat/completions)
期望的 Payload 格式：Gemini 格式（functionCall、role: model）
但是，当前代码发送的是 OpenAI 格式（tool_calls、role: assistant）

3. 为什么之前的修改都失败了？
尝试 1：添加 thought_signature 到 OpenAI 格式
❌ 失败：vectorengine 不接受 tool_calls，它期望 functionCall
尝试 2：转换为 Gemini 格式
❌ 失败：可能因为转换不完整或其他问题（需要验证）
尝试 3：移除格式转换，保持 OpenAI 格式
❌ 失败：vectorengine 明确要求 functionCall（Gemini 格式）
4. 正确的解决方案
需要在 gemini-payload-thought-signature.ts 中添加格式转换逻辑：

转换 assistant 消息：

role: assistant → role: model
tool_calls → content: [{ functionCall: {...} }]
删除 content 字段（或设为空数组）
转换 tool 消息：

role: tool → role: function
content → content: [{ functionResponse: {...} }]
确保 thought_signature 在正确位置：

functionCall 对象内部
functionCall wrapper 外部
