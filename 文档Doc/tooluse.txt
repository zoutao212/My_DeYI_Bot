工具 Schema 的问题
{
  "name": "write",
  "parameters": {
    "type": "object",
    "required": ["content"],  // ❌ 只要求 content，不要求 path
    "properties": {
      "path": { ... },         // ⚠️ 可选
      "content": { ... },      // ✅ 必需
      "file_path": { ... }     // ⚠️ 可选（重复字段）
    }
  }
}
问题：

❌ required 只包含 content，不包含 path 或 file_path
⚠️ 同时存在 path 和 file_path 两个字段（混乱）
❌ LLM 不知道应该使用哪个字段
LLM 的困惑
LLM 看到这个 schema 后：

知道必须提供 content ✅
但不知道路径字段是必需的 ❌
不知道应该用 path 还是 file_path ❌
在 stopReason: "error" 的情况下，选择返回文本而不是调用工具 ❌
