# 任务分解功能使用指南

**版本**：v20260204_1  
**最后更新**：2026-02-04

---

## 概述

任务分解功能允许 LLM 将复杂的任务自动分解成多个子任务，并通过队列系统依次执行。这对于处理大量内容生成、数据处理、多步骤任务等场景非常有用。

---

## 什么时候使用任务分解？

### ✅ 适合使用任务分解的场景

1. **大量内容生成**
   - 生成 10000 字的文章 → 分解成 5 个 2000 字的子任务
   - 生成 100 个产品描述 → 分解成 10 个批次，每批 10 个

2. **大量数据处理**
   - 总结 100 万字的电子书 → 分解成多个章节的子任务
   - 分析 1000 个文件 → 分解成多个批次的子任务

3. **多步骤任务**
   - 先读取文件，再分析内容，最后生成报告 → 分解成 3 个步骤的子任务
   - 先搜索信息，再整理数据，最后生成总结 → 分解成 3 个步骤的子任务

4. **并行处理**
   - 处理多个文件 → 为每个文件创建一个子任务
   - 生成多个图表 → 为每个图表创建一个子任务

### ❌ 不适合使用任务分解的场景

1. **简单任务**
   - 生成一段 200 字的文本 → 直接生成，不需要分解
   - 读取一个文件 → 直接读取，不需要分解

2. **需要上下文连贯的任务**
   - 生成一个连贯的故事 → 如果分解成多个子任务，可能会导致故事不连贯
   - 生成一个完整的代码文件 → 如果分解成多个子任务，可能会导致代码结构混乱

## 如何使用任务分解？

### 1. 用户发起请求

用户发送一个复杂的任务请求：

```
用户: 请帮我生成一个 10000 字的科幻小说
```

### 2. LLM 分解任务

LLM 识别到这是一个复杂任务，主动使用 `enqueue_task` 工具分解任务：

```typescript
// 调用 enqueue_task 5 次
enqueue_task({
  prompt: "请生成科幻小说的第 1-2000 字，包括开头和人物介绍",
  summary: "生成小说第 1-2000 字"
})

enqueue_task({
  prompt: "请生成科幻小说的第 2001-4000 字，继续故事发展",
  summary: "生成小说第 2001-4000 字"
})

// ... 继续调用 3 次
```

### 3. LLM 回复确认

LLM 向用户回复确认信息：

```
✅ 任务已分解完成！

📋 任务看板
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
🎯 主任务: 请帮我生成一个 10000 字的科幻小说
📝 子任务: 5 个
   ✅ 已完成: 0
   🔄 进行中: 0
   ⏳ 待执行: 5
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

💾 任务树保存位置: ~/.clawdbot/tasks/{sessionId}/TASK_TREE.json
📊 查看任务看板: 使用 show_task_board 工具

系统将自动执行这些子任务，每个子任务完成后会单独回复。
```

### 4. 系统自动执行子任务

系统会自动从队列中取出子任务，依次执行：

```
→ 执行任务 1: LLM 生成第 1-2000 字
→ 执行任务 2: LLM 生成第 2001-4000 字
→ 执行任务 3: LLM 生成第 4001-6000 字
→ 执行任务 4: LLM 生成第 6001-8000 字
→ 执行任务 5: LLM 生成第 8001-10000 字
```

每个子任务完成后，LLM 会单独回复用户。

## 任务树存储位置

### 存储路径

任务树会自动保存到以下位置：

```
~/.clawdbot/tasks/{sessionId}/
  ├── TASK_TREE.json          # 任务树主文件
  ├── TASK_TREE.json.bak      # 任务树备份文件
  ├── TASK_TREE.md            # 任务树 Markdown 格式
  ├── checkpoints/            # 检查点目录
  │   ├── {checkpointId}.json
  │   └── ...
  ├── failures.log            # 失败日志
  └── errors.log              # 错误日志
```

### 查看任务树

你可以使用以下方法查看任务树：

1. **使用 `show_task_board` 工具**（推荐）：
   ```typescript
   show_task_board()
   ```
   这会生成一个可视化的任务看板，展示所有子任务的状态。

2. **直接读取 JSON 文件**：
   ```bash
   cat ~/.clawdbot/tasks/{sessionId}/TASK_TREE.json
   ```

3. **读取 Markdown 文件**：
   ```bash
   cat ~/.clawdbot/tasks/{sessionId}/TASK_TREE.md
   ```

## 断点恢复

### 自动恢复

如果系统在执行任务过程中崩溃或被强制终止，系统会在下次启动时自动检查未完成的任务树，并尝试恢复：

```
[agent-runner] 🔍 Found unfinished tasks for session: {sessionId}
[agent-runner] ✅ Task tree recovered: {taskTreeId}
[agent-runner] ✅ Re-executed 3 interrupted tasks
```

### 检查点

系统会在以下时机自动创建检查点：

1. 每个子任务完成后
2. 每 5 个子任务完成后
3. 任务树状态发生重大变化时

检查点会保存到 `~/.clawdbot/tasks/{sessionId}/checkpoints/` 目录。

## 重要规则

### ✅ 正确使用

1. **用户直接要求时**：可以调用 `enqueue_task` 加入多个任务
   ```
   用户: 请生成 5 段内容
   → ✅ 正确：调用 enqueue_task 5 次，然后回复确认
   ```

2. **提供清晰的 prompt**：每个子任务的 prompt 应该清晰、具体、可执行
   ```typescript
   enqueue_task({
     prompt: "请生成科幻小说的第 1-2000 字，包括开头和人物介绍",
     summary: "生成小说第 1-2000 字"
   })
   ```

3. **提供简短的 summary**：每个子任务的 summary 应该简短地描述任务的目标
   ```typescript
   summary: "生成小说第 1-2000 字"
   ```

### ❌ 错误使用

1. **执行队列任务时调用 `enqueue_task`**：
   ```
   队列任务: 请生成第 1 段内容
   → ❌ 错误：调用 enqueue_task 生成更多任务（会导致无限循环）
   → ✅ 正确：直接生成第 1 段内容
   ```

2. **重复调用相同的工具**：
   ```
   → ❌ 错误：重复调用 enqueue_task 创建相同的任务
   → ✅ 正确：检查任务是否已经创建，避免重复
   ```

## 常见问题

### Q1: 任务树文件在哪里？

**A**: 任务树文件保存在 `~/.clawdbot/tasks/{sessionId}/TASK_TREE.json`。

你可以使用 `show_task_board` 工具查看任务树的可视化展示。

### Q2: 如何查看任务进度？

**A**: 使用 `show_task_board` 工具：

```typescript
show_task_board()
```

这会生成一个可视化的任务看板，展示所有子任务的状态。

### Q3: 如果系统崩溃了，任务会丢失吗？

**A**: 不会。系统会自动保存任务树到磁盘，并在下次启动时自动恢复未完成的任务。

### Q4: 如何手动恢复任务？

**A**: 系统会在启动时自动检查未完成的任务树，并尝试恢复。你不需要手动操作。

如果需要手动恢复，可以读取任务树文件：

```bash
cat ~/.clawdbot/tasks/{sessionId}/TASK_TREE.json
```

### Q5: 任务分解会影响性能吗？

**A**: 不会。任务分解只是将一个大任务拆分成多个小任务，每个小任务单独执行。这不会增加总的执行时间，反而可能提高效率（因为可以并行处理）。

---

## 总结

任务分解功能是 Clawdbot 的一个强大特性，它允许 LLM 自动将复杂的任务分解成多个子任务，并通过队列系统依次执行。

**关键要点**：
- ✅ 适合处理大量内容生成、数据处理、多步骤任务等场景
- ✅ 任务树会自动保存到磁盘，支持断点恢复
- ✅ 使用 `show_task_board` 工具查看任务进度
- ❌ 不要在执行队列任务时调用 `enqueue_task`（会导致无限循环）
