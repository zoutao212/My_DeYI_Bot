# 主动检索增强模块集成指南

## 已完成的修改

### 1. 导入语句已添加

在 `followup-runner.ts` 顶部已添加:

```typescript
import { proactiveRetrieval } from "../../agents/proactive-retrieval.js";
```

### 2. 需要手动添加的代码

在 `followup-runner.ts` 的第 919-935 行之间 (经验池注入之后，子任务记忆注入之前)，添加以下代码:

```typescript
// 🆕 主动检索增强：在用户消息进入 prompt 前，主动从多维度检索相关信息
// 检索维度：记忆系统 + 小说文本 + ToolCall 2.0 工具定义
let proactiveRetrievalCtx = "";
if (shouldInjectHeavyContext && queued.run.workspaceDir) {
  try {
    const retrievalResult = await proactiveRetrieval(config, {
      userMessage: queued.prompt,
      sessionId: taskTree?.id,
      sessionKey: queued.run.sessionKey,
      maxSnippets: 6,
      minScore: 0.35,
      enableMemory: true,
      enableNovel: true,
      enableAgentDef: false, // 暂时禁用，避免 prompt 膨胀
      enableToolDefs: true,
    });
    
    if (retrievalResult.formattedContext) {
      proactiveRetrievalCtx = retrievalResult.formattedContext;
      console.log(
        `[followup-runner] 🚀 主动检索完成：${retrievalResult.snippets.length} snippets, ` +
        `${retrievalResult.durationMs}ms, keywords=[${retrievalResult.extractedKeywords.slice(0, 5).join(", ")}]`,
      );
    }
  } catch (err) {
    console.debug(`主动检索失败 (不阻塞): ${err}`);
  }
}
```

### 3. 需要将 proactiveRetrievalCtx 注入到 extraSystemPrompt

在 `followup-runner.ts` 的第 1530 行左右，找到以下代码:

```typescript
const combined = [base, siblingCtx, novelReferenceCtxA, textEtlReferenceCtxA, parentGoalCtx, subTaskMemoryCtx, novelReferenceCtxB, textEtlReferenceCtxB, persistInstruction, blueprintCtx, novelReferenceCtxC, chapterOutlineCtx, experienceHint ? `\n\n${experienceHint}` : ""].filter(Boolean).join("");

修改为:

const combined = [base, proactiveRetrievalCtx, siblingCtx, novelReferenceCtxA, textEtlReferenceCtxA, parentGoalCtx, subTaskMemoryCtx, novelReferenceCtxB, textEtlReferenceCtxB, persistInstruction, blueprintCtx, novelReferenceCtxC, chapterOutlineCtx, experienceHint ? `\n\n${experienceHint}` : ""].filter(Boolean).join("");
```

注意在 `base` 后面添加了 `proactiveRetrievalCtx`。

## 配置选项

可以通过环境变量调整主动检索行为:

```bash
# 最大片段数
CLAWDBOT_PROACTIVE_MAX_SNIPPETS=6

# 最低分数阈值
CLAWDBOT_PROACTIVE_MIN_SCORE=0.35

# 启用/禁用通道
CLAWDBOT_PROACTIVE_ENABLE_MEMORY=1
CLAWDBOT_PROACTIVE_ENABLE_NOVEL=1
CLAWDBOT_PROACTIVE_ENABLE_TOOL_DEFS=1

# 超时保护 (毫秒)
CLAWDBOT_PROACTIVE_TIMEOUT_MS=3000
```
