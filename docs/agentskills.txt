Clawdbot Skills 系统架构
调用图谱：

用户消息 → Agent Runner → System Prompt 构建
                              ↓
                    buildWorkspaceSkillsPrompt()
                              ↓
                    loadSkillEntries() 从多个目录加载
                              ↓
                    ┌─────────────────────────────────────┐
                    │ 优先级（低→高）：                    │
                    │ 1. extraDirs (config.skills.load)   │
                    │ 2. bundledSkillsDir (skills/)       │
                    │ 3. managedSkillsDir (~/.clawdbot/)  │
                    │ 4. workspaceSkillsDir (项目/skills/)│
                    └─────────────────────────────────────┘
                              ↓
                    formatSkillsForPrompt() → 注入 System Prompt
                              ↓
                    LLM 看到 <available_skills> 列表
                              ↓
                    LLM 判断任务匹配 → 用 read 工具读取 SKILL.md
                              ↓
                    LLM 按 SKILL.md 指引执行（CLI/Python API）
关键发现：

Skills 是纯文档驱动的，不依赖 tool call / function call
LLM 通过 read 工具读取 SKILL.md，然后按文档指引执行 shell 命令
这正好解决了你说的"中转模型不支持 tool call"的问题
