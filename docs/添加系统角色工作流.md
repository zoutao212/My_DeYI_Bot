# 添加系统角色工作流指南

> 本文档基于德默泽尔（Demerzel）角色集成的实际经验总结，适用于向 Clawdbot 系统添加新的订制角色。

---

## 前置条件

- TypeScript 编译通过（`tsc --noEmit`）
- 了解现有角色目录结构（参考 `clawd/characters/` 下的 lina / demerzel / lisi）

---

## 第一步：创建角色文件（用户目录）

在用户的 clawd 目录下创建角色子目录：

```
C:\Users\{user}\clawd\characters\{角色名}\
├── config.json          # 角色配置（必需）
├── persona.md           # 灵魂定义 / 人格核心（必需）
├── profile.md           # 资产档案总览（必需）
├── prompts/
│   └── system.md        # 系统指令模板（必需）
├── knowledge/           # 知识库文件（可选，按需填充）
│   ├── body-blueprint.md
│   ├── sensory-archive.md
│   └── ...
├── scenes/              # 场景库（可选）
│   ├── genesis.md
│   └── ...
└── memory/
    ├── core-memories.md  # 核心记忆（必需）
    └── sessions/         # 会话归档目录
        └── .gitkeep
```

## 第二步：修改代码文件（4 个文件）

### 2.1 character-service.ts — 类型扩展

**文件**：`src/agents/pipeline/characters/character-service.ts`

确认 `FullCharacterConfig.type` 联合类型包含新角色类型。当前已支持：

```typescript
type: "system-persona" | "virtual-character" | "custom-aiji";
```

> 如果角色类型是 `custom-aiji`，无需修改此文件。新增角色类型才需要扩展。

### 2.2 register.ts — 硬编码角色识别

**文件**：`src/agents/pipeline/register.ts`

在 `registerPipelinePlugin` 函数的角色检测 `if-else` 链中添加新角色的匹配块：

```typescript
// 检测是否提到{角色名}
else if (userMessage.includes("中文名") || userMessage.includes("别名") || userMessage.includes("英文名")) {
  detectedCharacter = "角色英文ID";
  log.info("🔵 [Pipeline] Detected character: {id} ({显示名})");
}
```

**位置**：在现有角色检测链之后、丽丝检测之前（按角色优先级排序）。

> **关键**：`includes` 匹配的关键词应该是**最高频使用的称呼**，避免与其他角色关键词冲突。"爱姬"等共享关键词不要放在多个角色的检测中，否则只有第一个匹配的生效。

### 2.3 intent-analyzer.ts — 回退配置

**文件**：`src/agents/pipeline/intent-analyzer.ts`

在 `loadCharacterRecognitionConfig()` 方法的回退硬编码数组中添加新角色条目：

```typescript
{
  id: "角色英文ID",
  displayName: "中文显示名",
  isSystemPersona: false,
  recognition: {
    names: ["中文名", "英文名", "别名1", "别名2"],
    triggers: ["触发词1", "触发词2"],
    contexts: ["上下文关键词1", "上下文关键词2"],
  },
},
```

> 此配置在 `CharacterService` 动态加载失败时作为回退使用。

### 2.4 P89 记忆写入引导 — 4 个文件

需要在记忆写入引导的目录列表中添加新角色的记忆路径。

#### system-prompt.l10n.en.ts（类型定义 + 英文值）

1. 在 `SystemPromptL10n` 类型定义中新增字段：
   ```typescript
   memoryWriteHintDirChar{PascalCaseName}: string;
   ```

2. 在 `SYSTEM_PROMPT_L10N_EN` 对象中添加英文值：
   ```typescript
   memoryWriteHintDirChar{PascalCaseName}: "- Character memory: characters/{id}/memory/ ({DisplayName}, includes core-memories.md)",
   ```

#### system-prompt.l10n.zh.ts（中文值）

```typescript
memoryWriteHintDirChar{PascalCaseName}: "- 角色记忆：characters/{id}/memory/（{中文名}，含 core-memories.md）",
```

#### system-prompt.l10n.minimal.zh.ts（精简版，置空）

```typescript
memoryWriteHintDirChar{PascalCaseName}: "",
```

#### system-prompt.ts（buildMemoryWriteHint 函数）

在 `buildMemoryWriteHint()` 函数的返回数组中添加：

```typescript
l10n.memoryWriteHintDirChar{PascalCaseName},
```

## 第三步：同步仓库模板目录

将角色文件同步到仓库中的两个模板目录（供新用户初始化使用）：

```
clawd/characters/{角色名}/
├── config.json
├── persona.md
├── profile.md
├── prompts/system.md
└── memory/core-memories.md

user1/characters/{角色名}/
└── (同上)
```

> `user1/` 是开发/测试用的用户目录镜像，保持同步。

## 第五步：功能验证

### 唤醒测试

向系统发送包含角色关键词的消息，验证：
1. `register.ts` 硬编码识别是否命中
2. 日志中输出 `Detected character: {id}`
3. 角色的 `persona.md` / `profile.md` / `prompts/system.md` 被正确加载

### 记忆写入测试

发送"写入{角色名}记忆"类消息，验证 P89 引导中包含新角色的记忆路径。
