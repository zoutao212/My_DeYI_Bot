# Steering 文件索引

这些文件是 AI 助手的"记忆"，帮助保持协作的连续性。

---

## 文件说明

| 文件 | 用途 | 更新时机 |
|------|------|----------|
| `always/work_style‌.md` | 协作风格和沟通方式 | 了解用户习惯后更新 |
| `always/always_workflow‌.md` | 协作风格和沟通方式 | 了解用户习惯后更新 |
| `know.md` | 项目知识和开发规范 | 配置变更、规范调整时更新 |
| `product.md` | 产品概述和核心概念 | 产品定位变化时更新 |
| `structure.md` | 项目结构和数据库设计 | 架构变更、新增表时更新 |
| `tech.md` | 技术栈和常用命令 | 技术选型变化时更新 |
| `lessons-learned.md` | 踩过的坑和经验教训 | 遇到问题解决后记录 |

---

## 使用原则

### 什么时候更新
- 发现新的项目知识 → 更新对应文件
- 踩了坑并解决 → 记录到 `lessons-learned.md`
- 用户告知新的偏好 → 更新 `dev-experience.md`
- 架构/数据库变更 → 更新 `structure.md`

### 什么不该放这里
- 临时的任务状态（用 spec 文档）
- 具体的代码实现（放代码注释）
- 一次性的操作记录（不需要记录）

---

## 文件分类

### 项目认知（关于项目本身）
- `product.md` - 产品是什么
- `structure.md` - 代码怎么组织
- `tech.md` - 用什么技术
- `know.md` - 开发规范和配置

### 协作认知（关于如何合作）
- `dev-experience.md` - 沟通风格和习惯
- `lessons-learned.md` - 经验教训
