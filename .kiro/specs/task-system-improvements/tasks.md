## 注意事项

### 实施原则

1. **向后兼容**: 所有新字段使用可选类型,确保旧代码无需修改
2. **渐进式实现**: 每个阶段独立可交付,可以逐步部署
3. **充分测试**: 每个功能都要有对应的测试
4. **性能优先**: 确保新功能不影响系统性能

### 测试策略

- **单元测试**: 测试各个组件的独立功能
- **集成测试**: 测试组件之间的协作
- **性能测试**: 验证性能指标
- **快照测试**: 验证 HTML 输出的稳定性
