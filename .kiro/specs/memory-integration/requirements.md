# 记忆系统集成需求文档

## 1. 项目概述

### 1.1 背景
多层 Agent 架构已基本完成（92.5% 完成率），但记忆系统集成缺失，导致：
- 对话前记忆填充只是占位符
- 对话后总结归档未实现
- Agent 无法利用历史对话和长期记忆
- 对话连贯性和个性化体验受影响

### 1.2 目标
将现有的记忆系统（`src/memory/`）与多层架构（管家层、虚拟世界层）完整集成，实现：
- 对话前自动检索相关记忆并注入上下文
- 对话后自动总结并归档到长期记忆
- 支持多层架构的不同层次使用记忆系统
- 提供统一的记忆管理接口

### 1.3 优先级
⭐⭐⭐⭐⭐ 最高优先级（影响对话连贯性和个性化）

### 1.4 预计时间
2-3 小时

## 2. 用户故事

### 2.1 对话前记忆填充

**作为** 用户  
**我希望** Agent 在对话前自动检索相关的历史记忆  
**以便** Agent 能够理解上下文，提供连贯的对话体验

**验收标准：**
- [ ] 管家层在处理消息前，自动调用记忆检索
- [ ] 检索结果注入到对话上下文中
- [ ] 检索失败不影响对话流程（降级处理）
- [ ] 检索结果包含相关性评分和时间戳
- [ ] 支持配置检索参数（最大结果数、最小相关性分数）

### 2.2 对话后总结归档

**作为** 用户  
**我希望** Agent 在对话后自动总结并归档到长期记忆  
**以便** 未来的对话可以利用这些记忆

**验收标准：**
- [ ] 管家层在对话结束后，自动生成总结
- [ ] 总结包含关键信息（任务目标、关键操作、决策、问题）
- [ ] 总结异步归档到记忆系统（不阻塞对话）
- [ ] 归档失败记录日志但不影响用户体验
- [ ] 支持配置总结策略（何时总结、总结内容）

### 2.3 虚拟世界层记忆支持

**作为** 虚拟世界层的角色  
**我希望** 能够访问与角色相关的记忆  
**以便** 提供更真实的角色扮演体验

**验收标准：**
- [ ] 虚拟世界层可以检索角色相关的记忆
- [ ] 记忆检索不暴露技术细节（符合角色设定）
- [ ] 支持角色特定的记忆过滤
- [ ] 记忆注入不破坏角色人格

### 2.4 记忆管理接口

**作为** 开发者  
**我希望** 有统一的记忆管理接口  
**以便** 各层可以方便地使用记忆系统

**验收标准：**
- [ ] 提供统一的记忆检索接口
- [ ] 提供统一的记忆归档接口
- [ ] 支持不同层次的记忆隔离
- [ ] 提供记忆状态查询接口
- [ ] 提供记忆配置接口

## 3. 功能需求

### 3.1 记忆检索服务

#### 3.1.1 检索接口
- 输入：查询文本、上下文信息、检索参数
- 输出：相关记忆列表（包含路径、内容、相关性分数、时间戳）
- 支持混合检索（向量 + 关键词）
- 支持来源过滤（memory 文件 vs sessions 文件）

#### 3.1.2 上下文注入
- 将检索结果格式化为上下文
- 支持不同格式（System Prompt、消息历史）
- 支持相关性排序和截断
- 支持记忆去重

### 3.2 记忆归档服务

#### 3.2.1 总结生成
- 使用现有的 `session-summary.ts` 生成总结
- 提取关键信息（任务目标、操作、决策、问题）
- 支持增量总结（只总结新内容）
- 支持自定义总结策略

#### 3.2.2 归档存储
- 将总结写入 memory 文件
- 支持自动分类（按日期、主题、项目）
- 支持归档格式配置（Markdown、JSON）
- 支持归档路径配置

### 3.3 层次集成

#### 3.3.1 管家层集成
- 在 `beforeConversation` 中调用记忆检索
- 在 `afterConversation` 中调用记忆归档
- 支持记忆检索失败降级
- 支持记忆归档异步执行

#### 3.3.2 虚拟世界层集成
- 提供角色特定的记忆检索
- 过滤技术细节（符合角色设定）
- 支持角色记忆隔离
- 支持角色记忆格式化

### 3.4 配置管理

#### 3.4.1 检索配置
- 最大结果数（默认：5）
- 最小相关性分数（默认：0.7）
- 检索来源（memory、sessions、both）
- 检索超时时间（默认：5 秒）

#### 3.4.2 归档配置
- 归档策略（always、on-demand、threshold）
- 归档路径（默认：`memory/sessions/`）
- 归档格式（markdown、json）
- 归档频率（每次对话、每 N 轮、每 N 分钟）

## 4. 非功能需求

### 4.1 性能
- 记忆检索响应时间 < 1 秒（P95）
- 记忆归档不阻塞对话（异步执行）
- 支持记忆缓存（减少重复检索）
- 支持批量归档（减少 I/O）

### 4.2 可靠性
- 记忆检索失败不影响对话流程
- 记忆归档失败记录日志
- 支持记忆系统降级（禁用记忆功能）
- 支持记忆数据备份

### 4.3 可维护性
- 记忆接口清晰易用
- 记忆配置灵活可扩展
- 记忆日志详细可追踪
- 记忆测试覆盖完整

### 4.4 兼容性
- 兼容现有的记忆系统（`src/memory/`）
- 兼容现有的会话总结（`src/agents/session-summary.ts`）
- 兼容多层架构的各层
- 兼容现有的配置系统

## 6. 风险与缓解

### 6.1 风险
1. **记忆检索性能问题** - 检索时间过长影响对话体验
2. **记忆归档失败** - 归档失败导致记忆丢失
3. **记忆注入格式问题** - 格式不当影响 LLM 理解
4. **记忆系统不可用** - 记忆系统故障影响对话

### 6.2 缓解措施
1. **性能问题** - 设置检索超时，使用缓存，异步预加载
2. **归档失败** - 异步执行，失败重试，记录日志
3. **格式问题** - 使用标准格式，测试验证，提供示例
4. **系统不可用** - 降级处理，禁用记忆功能，记录日志
