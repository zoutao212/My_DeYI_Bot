# 通用文件发送系统 - 需求文档

**功能名称**：universal-file-sending  
**创建日期**：2026-02-06  
**状态**：待实施

---

## 1. 功能概述

将文件发送能力固化为系统核心功能，支持 LLM 在任意对话场景中主动发送文件到用户的聊天频道。

### 1.1 核心价值

- **通用性**：适用于任意对话场景，不限于特定功能
- **主动性**：LLM 可以根据用户需求主动发送文件
- **多频道**：统一接口支持多种聊天平台
- **易集成**：与现有功能（如写作任务）无缝集成

## 3. 功能需求

### 3.1 核心功能

#### 3.1.1 send_file 工具

**需求**：创建新的 Agent 工具，允许 LLM 发送文件

**参数**：
- `filePath`（必需）：文件路径（绝对路径或相对路径）
- `caption`（可选）：文件说明

**行为**：
1. 解析文件路径（支持绝对路径和相对路径）
2. 检查文件是否存在
3. 获取会话信息（频道、用户 ID、线程 ID）
4. 根据频道类型调用对应的发送方法
5. 返回发送结果

**返回值**：
```typescript
{
  success: boolean;
  message?: string;
  error?: string;
  fileName?: string;
  fileSize?: number;
  channel?: string;
}
```

#### 3.1.2 Telegram 文件发送

**需求**：实现 Telegram 频道的文件发送

**功能**：
- 使用 grammy 库发送文件
- 支持文件说明（caption）
- 支持线程回复（message_thread_id）
- 文件大小限制：50MB

#### 3.1.3 Discord 文件发送

**需求**：实现 Discord 频道的文件发送

**功能**：
- 使用 Discord API 发送文件
- 支持文件说明
- 文件大小限制：8MB

#### 3.1.4 Slack 文件发送

**需求**：实现 Slack 频道的文件发送

**功能**：
- 使用 Slack API 发送文件
- 支持文件说明
- 文件大小限制：1GB

#### 3.1.5 Web 网关文件展示

**需求**：在 Web 网关中展示文件

**功能**：
- 在聊天界面显示文件卡片
- 提供文件下载链接
- 显示文件信息（名称、大小、类型）

### 3.2 安全需求

#### 3.2.1 路径验证

**需求**：防止路径遍历攻击

**验收标准**：
- [ ] 只允许访问工作目录内的文件
- [ ] 只允许访问任务目录内的文件
- [ ] 拒绝访问系统敏感目录

#### 3.2.2 文件大小限制

**需求**：防止发送超大文件

**验收标准**：
- [ ] Telegram：最大 50MB
- [ ] Discord：最大 8MB
- [ ] Slack：最大 1GB
- [ ] 超过限制时返回清晰的错误消息

#### 3.2.3 文件类型检查

**需求**：防止发送危险文件

**允许的文件类型**：
- 文档：.txt, .md, .pdf, .docx, .xlsx
- 图片：.png, .jpg, .jpeg, .gif
- 压缩：.zip, .tar, .gz
- 数据：.json, .csv, .xml

**验收标准**：
- [ ] 只允许发送白名单内的文件类型
- [ ] 拒绝发送可执行文件（.exe, .sh, .bat）
- [ ] 拒绝发送脚本文件（.js, .py, .rb）

## 4. 非功能需求

### 4.1 性能需求

- 文件发送响应时间 < 5 秒（小文件）
- 文件发送响应时间 < 30 秒（大文件）
- 支持并发发送多个文件

### 4.2 可靠性需求

- 文件发送成功率 > 99%
- 发送失败时有清晰的错误消息
- 支持重试机制

### 4.3 可维护性需求

- 代码结构清晰，易于扩展新频道
- 完整的单元测试覆盖
- 详细的错误日志
