# 代码与注释一致性验证

> **模式类型**: 质量保证流程  
> **首次发现**: 2026-01-29  
> **版本**: v20260129_1

---

## 问题模式

### 典型现象

代码注释说"默认展开(true)",但实际初始值是"折叠(false)":

```typescript
// 注释说默认 true
const showFull = state.llmApprovalShowFullPayload !== false; // 默认 true

// 但实际初始值是 false
@state() llmApprovalShowFullPayload = false;
```

### 为什么会反复发生

1. **代码演进**: 修改了实现,但忘记更新注释
2. **期望 vs 现实**: 注释写的是"期望行为",但实现没跟上
3. **多人协作**: 不同人修改代码和注释,没有同步
4. **缺少验证**: 没有检查注释与实际行为是否一致

---

## 解决方案

### 交叉验证流程

修改代码时,必须执行以下验证:

```
1. 读取代码实现
   ↓
2. 读取注释说明
   ↓
3. 观察实际行为
   ↓
4. 三者是否一致?
   ├─ 是 → 通过
   └─ 否 → 以实际行为为准,修正代码或注释
```

### 具体步骤

#### 步骤 1: 识别不一致

**检查点:**
- 变量初始值 vs 注释说明
- 函数行为 vs 文档描述
- 配置默认值 vs 说明文档

**工具:**
```powershell
# 搜索"默认"相关的注释
Select-String -Path "**/*.ts" -Pattern "默认|default|初始" -Context 2,2

# 搜索状态初始化
Select-String -Path "**/*.ts" -Pattern "@state\(\)|useState|初始值" -Context 1,1
```

#### 步骤 2: 验证实际行为

**方法:**
1. 运行代码,观察实际行为
2. 检查日志输出
3. 使用调试工具查看变量值
4. 询问用户实际体验

#### 步骤 3: 修正不一致

**原则: 以实际行为为准**

| 情况 | 处理方式 |
|------|---------|
| 注释错误,代码正确 | 修正注释 |
| 代码错误,注释正确 | 修正代码 |
| 都错误 | 确认期望行为,同时修正 |
| 不确定 | 询问用户或查看历史记录 |

#### 步骤 4: 添加验证标记

修改后添加验证标记:

```typescript
// ✅ 已验证: 默认展开,初始值为 true (2026-01-29)
@state() llmApprovalShowFullPayload = true;
```

## 常见场景

### 场景 1: 布尔值默认状态

**问题:**
```typescript
// 注释说默认 true
const enabled = config.feature !== false; // 默认启用

// 但配置文件中是 false
feature: false
```

**验证:**
```powershell
# 1. 查看代码
Select-String -Path "src/**/*.ts" -Pattern "config.feature"

# 2. 查看配置
Get-Content "config.json" | Select-String "feature"

# 3. 运行测试
pnpm test -- feature
```

### 场景 2: 数值默认值

**问题:**
```typescript
// 注释说默认 1 次重试
const MAX_RETRIES = 3; // 最多 1 次重试
```

**验证:**
```typescript
// 添加断言测试
expect(MAX_RETRIES).toBe(1); // 如果注释是对的
// 或者
expect(MAX_RETRIES).toBe(3); // 如果代码是对的
```

### 场景 3: 字符串默认值

**问题:**
```typescript
// 注释说默认中文
const language = config.lang || 'en'; // 默认中文
```

**验证:**
```powershell
# 检查实际默认值
node -e "console.log(require('./config').lang || 'en')"
```

## 预防措施

### 1. 代码审查清单

修改代码时检查:
- [ ] 注释是否与代码一致?
- [ ] 默认值是否与说明一致?
- [ ] 文档是否需要更新?

### 2. 自动化测试

添加测试验证默认值:

```typescript
describe('默认值验证', () => {
  it('审批 UI 默认展开', () => {
    const state = new AppState();
    expect(state.llmApprovalShowFullPayload).toBe(true);
  });
  
  it('重试次数为 1', () => {
    expect(MAX_ATTEMPTS_PER_KEY).toBe(1);
  });
});
```

### 3. 文档同步

修改代码时同步更新:
- 代码注释
- API 文档
- 用户手册
- 配置说明
