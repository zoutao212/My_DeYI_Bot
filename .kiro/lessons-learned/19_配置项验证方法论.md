# 配置项验证方法论

> **核心原则：配置值必须对照代码中的 schema 验证，不要假设配置是正确的。**

---

## 问题模式

### 现象

- 修改了配置文件，但系统行为没有变化
- 配置看起来"正确"，但实际不生效
- 系统静默失败，没有报错信息

### 根本原因

**配置值不在代码 schema 支持列表中：**

1. **手动填写错误**：拼写错误、使用了旧版本的值、参考了错误的文档
2. **缺少验证**：没有对照代码中的 schema 验证配置值
3. **静默失败**：系统没有报错，fallback 到默认行为

### 实战案例

**案例 1：API 格式配置错误**

**配置文件（错误）：**
```json
{
  "models": {
    "providers": {
      "vectorengine": {
        "api": "gemini"  // ❌ 不在支持列表中
      }
    }
  }
}
```

**代码 schema（正确）：**
```typescript
// src/config/zod-schema.core.ts
export const ModelApiSchema = z.union([
  z.literal("openai-completions"),
  z.literal("openai-responses"),
  z.literal("anthropic-messages"),
  z.literal("google-generative-ai"),  // ✅ 正确的值
  z.literal("github-copilot"),
  z.literal("bedrock-converse-stream"),
]);
```

**结果：**
- 配置中写的是 `"gemini"`
- 代码中支持的是 `"google-generative-ai"`
- 系统静默失败，fallback 到默认的 `"openai-completions"`
- 导致系统行为与预期不符

## 标准验证流程

### 第一步：找到配置项对应的 schema 定义

**常见位置：**
- `src/config/zod-schema.*.ts` - 配置 schema 定义
- `src/types/*.ts` - 类型定义
- `src/*/config.ts` - 模块配置定义

**搜索方法：**
```powershell
# 搜索 schema 定义
grepSearch -query "配置项名称.*Schema" -includePattern "src/config/**/*.ts"

# 搜索类型定义
grepSearch -query "type.*配置项名称|interface.*配置项名称" -includePattern "src/**/*.ts"
```

**示例：**
```powershell
# 搜索 ModelApi 的 schema
grepSearch -query "ModelApiSchema" -includePattern "src/config/**/*.ts"
```

### 第二步：对照 schema 验证配置值

**检查点：**

1. **枚举值（z.literal / z.enum）：**
   ```typescript
   // schema 定义
   z.union([
     z.literal("value1"),
     z.literal("value2"),
     z.literal("value3"),
   ])
   
   // 配置值必须是其中之一
   ```

2. **字符串格式（z.string）：**
   ```typescript
   // schema 定义
   z.string().min(1).max(100)
   
   // 配置值必须是非空字符串，长度 ≤ 100
   ```

3. **数字范围（z.number）：**
   ```typescript
   // schema 定义
   z.number().min(0).max(100)
   
   // 配置值必须是 0-100 之间的数字
   ```

4. **对象结构（z.object）：**
   ```typescript
   // schema 定义
   z.object({
     field1: z.string(),
     field2: z.number().optional(),
   })
   
   // 配置值必须包含 field1，field2 可选
   ```

### 第三步：查找正确的配置值

**方法 1：查看 schema 定义**
```typescript
// 直接从 schema 中找到所有支持的值
export const ModelApiSchema = z.union([
  z.literal("openai-completions"),     // ✅ 支持
  z.literal("openai-responses"),       // ✅ 支持
  z.literal("google-generative-ai"),   // ✅ 支持
]);
```

**方法 2：搜索代码中的使用示例**
```powershell
# 搜索配置项的使用
grepSearch -query "api.*=.*\"" -includePattern "src/**/*.ts"

# 搜索测试用例中的示例
grepSearch -query "api.*:" -includePattern "**/*.test.ts"
```

**方法 3：查看文档或注释**
```typescript
// 有些 schema 会有注释说明
export const ModelApiSchema = z.union([
  z.literal("openai-completions"),  // OpenAI Completions API
  z.literal("google-generative-ai"), // Google Gemini API
]);
```

### 第四步：修改配置并验证

**修改配置：**
```powershell
# 备份配置
Copy-Item "~/.clawdbot/clawdbot.json" "~/.clawdbot/clawdbot.json.backup_$(Get-Date -Format 'yyyyMMdd_HHmmss')"

# 修改配置（手动编辑或脚本）
$config = Get-Content "~/.clawdbot/clawdbot.json" -Raw -Encoding UTF8 | ConvertFrom-Json
$config.models.providers.vectorengine.api = "google-generative-ai"  # 改为正确的值
$config | ConvertTo-Json -Depth 20 | Out-File "~/.clawdbot/clawdbot.json" -Encoding UTF8
```

**验证配置：**
```powershell
# 读取配置并验证
$config = Get-Content "~/.clawdbot/clawdbot.json" -Raw -Encoding UTF8 | ConvertFrom-Json
Write-Host "api: $($config.models.providers.vectorengine.api)"

# 确认值是否正确
if ($config.models.providers.vectorengine.api -eq "google-generative-ai") {
    Write-Host "✅ 配置正确"
} else {
    Write-Host "❌ 配置错误"
}
```

### 第五步：重启服务并验证行为

**重启服务：**
```powershell
# 停止服务
.\Gateway-Service-Stop.cmd

# 启动服务
.\Gateway-Service-Start.cmd
```

**验证行为：**
1. 查看日志，确认使用了正确的配置值
2. 测试功能，确认行为符合预期
3. 如果仍然不符合预期，检查是否有其他配置覆盖

## 预防措施

### 1. 使用配置验证工具

**如果项目有配置验证工具，优先使用：**
```powershell
# 验证配置文件
clawdbot config validate

# 查看配置
clawdbot config show
```

### 2. 参考官方文档

**优先参考官方文档，而不是猜测：**
- 查看 `docs/` 目录
- 查看 `README.md`
- 查看代码注释

### 3. 查看测试用例

**测试用例通常包含正确的配置示例：**
```powershell
# 搜索测试用例
grepSearch -query "配置项名称" -includePattern "**/*.test.ts"
```

### 4. 使用类型检查

**如果配置文件是 TypeScript，使用类型检查：**
```powershell
# 类型检查
pnpm tsc --noEmit
```

## 调试技巧

### 技巧 1：添加日志验证

**在代码中添加日志，确认配置值：**
```typescript
console.log("配置值:", config.api);
console.log("期望值:", "google-generative-ai");
console.log("是否匹配:", config.api === "google-generative-ai");
```

### 技巧 2：使用断点调试

**在配置读取处设置断点：**
```typescript
// 在这里设置断点
const api = config.models.providers.vectorengine.api;
console.log("api:", api);
```

### 技巧 3：对比默认配置

**对比默认配置和当前配置：**
```powershell
# 查看默认配置
clawdbot config show --default

# 查看当前配置
clawdbot config show

# 对比差异
diff default.json current.json
```

## 检查清单

配置项验证时，必须检查：

- [ ] **找到 schema 定义**：确认配置项对应的 schema 位置
- [ ] **对照 schema 验证**：确认配置值在支持列表中
- [ ] **查找正确的值**：如果配置值错误，找到正确的值
- [ ] **修改配置**：修改配置文件并备份
- [ ] **验证配置**：读取配置并确认修改成功
- [ ] **重启服务**：重启服务使配置生效
- [ ] **验证行为**：测试功能，确认行为符合预期
- [ ] **查看日志**：确认日志中使用了正确的配置值

---

## 关键教训

1. **不要假设配置是正确的**
   - 配置值可能拼写错误
   - 配置值可能是旧版本的
   - 配置值可能不在支持列表中

2. **必须对照 schema 验证**
   - 找到配置项对应的 schema 定义
   - 对照 schema 中的枚举值/格式要求
   - 确认配置值在支持列表中

3. **系统可能静默失败**
   - 配置错误时，系统可能不报错
   - 系统可能 fallback 到默认行为
   - 导致行为与预期不符

4. **验证要双向**
   - 不仅要验证配置文件
   - 还要验证运行时行为
   - 两者结合才能确认配置生效
