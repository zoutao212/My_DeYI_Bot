# 工具增强的包装模式

> **背景**：当现有工具功能不足时，如何在不破坏现有功能的前提下增强工具

---

## 问题现象

### 典型场景

1. **功能缺失**：现有工具不支持某些场景
   - 示例：`read` 工具不支持非 UTF-8 编码

2. **外部依赖限制**：依赖的外部库功能有限
   - 示例：`@mariozechner/pi-coding-agent` 的 `createReadTool` 只支持 UTF-8

3. **需要保持兼容**：不能破坏现有代码
   - 示例：现有代码都使用 `read({ path: "..." })`，不能改变这个接口

### 错误特征

- 用户报告工具无法处理某些情况
- 工具在特定场景下失败或返回错误结果
- 需要手动处理工具无法处理的情况

---

## 根本原因

### 设计时未考虑

**问题链条**：
```
工具设计时只考虑常见场景
  ↓
未考虑边缘情况或特殊需求
  ↓
用户遇到特殊场景时工具失败
  ↓
需要增强工具
```

### 外部依赖限制

- 依赖的外部库功能有限
- 无法直接修改外部库代码
- 需要在包装层增强

### 向后兼容要求

- 现有代码依赖当前接口
- 不能破坏现有功能
- 需要平滑升级

---

## 解决方案：包装增强模式

### 核心原则

1. **包装而非替换**：在现有工具外层添加功能
2. **参数扩展**：添加可选参数，保持向后兼容
3. **智能回退**：新功能失败时回退到原有逻辑
4. **清晰提示**：告知用户使用的功能和可能的问题

### 实施步骤

#### 步骤 1：分析现有工具

**目标**：理解现有工具的接口和实现

**检查清单**：
- [ ] 现有工具的参数和返回值
- [ ] 现有工具的实现逻辑
- [ ] 现有工具的依赖关系
- [ ] 现有工具的使用场景

**示例**：
```typescript
// 现有工具
const baseTool = createReadTool(root);

// 接口
interface ReadParams {
  path: string;
}

// 返回值
interface ReadResult {
  content: Array<{ type: "text" | "image"; ... }>;
  details: unknown;
}
```

#### 步骤 2：设计增强功能

**目标**：设计新功能的接口和实现

**设计原则**：
- **可选参数**：新参数必须是可选的
- **默认行为**：不指定新参数时，行为与原工具一致
- **清晰语义**：参数名和值要清晰易懂

**示例**：
```typescript
// 增强后的接口
interface EnhancedReadParams {
  path: string;
  encoding?: "utf-8" | "gbk" | "gb2312" | "big5" | "shift_jis" | "auto";  // 新增
}

// 默认值
const defaultEncoding = "auto";
```

#### 步骤 3：实现包装层

**目标**：在现有工具外层添加增强功能

**实现模式**：
```typescript
export function createEnhancedTool(baseTool: Tool): Tool {
  // 1. 扩展 schema
  const enhancedSchema = {
    ...baseTool.parameters,
    properties: {
      ...baseTool.parameters.properties,
      newParam: {  // 新增参数
        type: "string",
        description: "...",
        enum: [...],
      },
    },
  };
  
  // 2. 包装 execute
  return {
    ...baseTool,
    parameters: enhancedSchema,
    execute: async (toolCallId, params, signal) => {
      // 提取新参数
      const newParam = params?.newParam ?? defaultValue;
      
      // 判断是否需要使用新功能
      if (shouldUseNewFeature(params, newParam)) {
        try {
          // 使用新功能
          return await executeNewFeature(params, newParam);
        } catch (err) {
          // 失败时回退到原有逻辑
          const result = await baseTool.execute(toolCallId, params, signal);
          
          // 添加警告信息
          return addWarning(result, err);
        }
      }
      
      // 默认使用原有逻辑
      return baseTool.execute(toolCallId, params, signal);
    },
  };
}
```

#### 步骤 4：实现智能回退

**目标**：新功能失败时不影响原有功能

**回退策略**：
```typescript
try {
  // 尝试使用新功能
  return await executeNewFeature(params);
} catch (err) {
  // 记录错误
  logWarning(`New feature failed: ${err}`);
  
  // 回退到原有逻辑
  const result = await baseTool.execute(toolCallId, params, signal);
  
  // 添加警告信息（可选）
  return {
    ...result,
    content: [
      {
        type: "text",
        text: `⚠️ Warning: ${err.message}\nFalling back to default behavior.\n\n`,
      },
      ...result.content,
    ],
  };
}
```

#### 步骤 5：验证向后兼容

**目标**：确保现有代码无需修改

**验证清单**：
- [ ] 不指定新参数时，行为与原工具一致
- [ ] 现有测试用例全部通过
- [ ] 现有代码无需修改
- [ ] 新参数是可选的

**测试**：
```typescript
// 测试 1：不指定新参数（向后兼容）
const result1 = await tool.execute("id", { path: "file.txt" });
// 预期：与原工具行为一致

// 测试 2：指定新参数（新功能）

// 预期：使用新功能

// 测试 3：新功能失败（回退）

// 预期：回退到原有逻辑，显示警告
```

## 实战案例：read 工具编码支持

### 问题

`read` 工具不支持非 UTF-8 编码，导致读取 GBK/GB2312 文件时出现乱码。

### 分析

1. **现有工具**：`createReadTool()` 来自外部库，只支持 UTF-8
2. **需求**：支持 GBK、GB2312、Big5、Shift_JIS 等编码
3. **约束**：不能修改外部库，必须保持向后兼容

### 实施

#### 步骤 1：分析现有工具

```typescript
// 现有工具
const baseTool = createReadTool(root);

// 接口
interface ReadParams {
  path: string;
}
```

#### 步骤 2：设计增强功能

```typescript
// 新增 encoding 参数
interface EnhancedReadParams {
  path: string;
  encoding?: "utf-8" | "gbk" | "gb2312" | "big5" | "shift_jis" | "auto";
}

// 默认值：auto（自动检测）
```

#### 步骤 3：实现包装层

```typescript
export function createClawdbotReadTool(base: AnyAgentTool): AnyAgentTool {
  // 1. 扩展 schema
  const enhancedSchema = {
    ...base.parameters,
    properties: {
      ...base.parameters.properties,
      encoding: {
        type: "string",
        description: "Text file encoding (utf-8, gbk, gb2312, big5, shift_jis, auto)",
        enum: ["utf-8", "gbk", "gb2312", "big5", "shift_jis", "auto"],
      },
    },
  };
  
  // 2. 包装 execute
  return {
    ...base,
    parameters: enhancedSchema,
    execute: async (toolCallId, params, signal) => {
      const encoding = params?.encoding ?? "auto";
      const filePath = params?.path;
      
      // 判断是否需要使用编码功能
      const isTextFile = filePath.match(/\.(txt|md|json|...)$/i);
      
      if (isTextFile && encoding !== "utf-8") {
        try {
          // 使用编码功能
          const content = await readFileWithEncoding(filePath, encoding);
          return {
            content: [{ type: "text", text: content }],
            details: { encoding, filePath },
          };
        } catch (err) {
          // 回退到原有逻辑
          const result = await base.execute(toolCallId, params, signal);
          
          // 添加警告
          return {
            ...result,
            content: [
              { type: "text", text: `⚠️ Warning: ${err}\n\n` },
              ...result.content,
            ],
          };
        }
      }
      
      // 默认使用原有逻辑
      return base.execute(toolCallId, params, signal);
    },
  };
}
```

#### 步骤 4：实现编码功能

```typescript
async function detectTextEncoding(filePath: string): Promise<string> {
  const encodings = ["utf-8", "gbk", "gb2312", "big5", "shift_jis"];
  const buffer = await fs.promises.readFile(filePath);
  
  for (const encoding of encodings) {
    try {
      const decoder = new TextDecoder(encoding, { fatal: true });
      const text = decoder.decode(buffer);
      
      if (!text.includes("\uFFFD")) {
        return encoding;
      }
    } catch {
      continue;
    }
  }
  
  return "utf-8";
}

async function readFileWithEncoding(
  filePath: string,
  encoding: string,
): Promise<string> {
  const buffer = await fs.promises.readFile(filePath);
  
  if (encoding === "auto") {
    encoding = await detectTextEncoding(filePath);
  }
  
  const decoder = new TextDecoder(encoding, { fatal: false });
  return decoder.decode(buffer);
}
```

#### 步骤 5：验证

```typescript

// 测试 2：新功能

// 预期：使用 GBK 编码读取

// 测试 3：自动检测
```

## 最佳实践

### 1. 参数设计

**原则**：
- 新参数必须是可选的
- 提供合理的默认值
- 参数名要清晰易懂

**示例**：
```typescript
// ✅ 好的设计
interface Params {
  path: string;           // 必需
  encoding?: string;      // 可选，默认 "auto"
}

// ❌ 不好的设计
interface Params {
  path: string;
  encoding: string;       // 必需，破坏向后兼容
}
```

### 2. 回退策略

**原则**：
- 新功能失败时不影响原有功能
- 提供清晰的错误信息
- 记录失败原因

**示例**：
```typescript
try {
  return await newFeature(params);
} catch (err) {
  // 记录错误
  logWarning(`New feature failed: ${err}`);
  
  // 回退
  const result = await originalFeature(params);
  
  // 添加警告
  return addWarning(result, `Failed to use new feature: ${err}`);
}
```

## 常见错误模式

### 错误 1：破坏向后兼容

```typescript
// ❌ 错误：新参数是必需的
interface Params {
  path: string;
  encoding: string;  // 必需
}

// ✅ 正确：新参数是可选的
interface Params {
  path: string;
  encoding?: string;  // 可选
}
```

### 错误 2：没有回退机制

```typescript
// ❌ 错误：新功能失败时抛出错误
if (useNewFeature) {
  return await newFeature(params);  // 可能失败
}

// ✅ 正确：失败时回退
if (useNewFeature) {
  try {
    return await newFeature(params);
  } catch (err) {
    return await originalFeature(params);
  }
}
```

### 错误 3：修改原有行为

```typescript
// ❌ 错误：改变默认行为
const encoding = params?.encoding ?? "gbk";  // 默认 GBK

// ✅ 正确：保持默认行为
const encoding = params?.encoding ?? "utf-8";  // 默认 UTF-8（原有行为）
```

## 适用场景

### 适合使用包装增强模式

- ✅ 需要添加新功能到现有工具
- ✅ 外部依赖功能有限，无法修改
- ✅ 必须保持向后兼容
- ✅ 新功能是可选的

### 不适合使用包装增强模式

- ❌ 需要完全重写工具
- ❌ 新功能与原有功能冲突
- ❌ 无法保持向后兼容
- ❌ 新功能是必需的
