## 问题模式

### 典型错误

**错误实现**: 把"所有请求"都计数,导致成功请求也被计入重试次数

```typescript
// ❌ 错误: 不区分成功和失败
const attempts = new Map<string, { firstAtMs: number; count: number }>();

// 每次请求都增加计数
attempts.set(key, { firstAtMs: now, count: current.count + 1 });

// 导致: 成功请求也被计入,很快就超过限制
```

**结果**: 
- 成功的请求被错误地计入重试次数
- 用户看到"已超过最大重试次数",但实际上请求是成功的
- 日志显示重试了 4 次,但配置是"最多 1 次"

---

## 正确设计

### 核心原则

**只计算失败次数,成功时清除计数**

```typescript
// ✅ 正确: 区分成功和失败
const attempts = new Map<string, { firstAtMs: number; failureCount: number }>();

async function executeRequest(key: string, request: Request): Promise<Response> {
  try {
    const response = await fetch(request);
    
    if (response.ok) {
      // 成功: 清除失败计数
      attempts.delete(key);
    } else {
      // 失败: 增加失败计数
      incrementFailureCount(key);
    }
    
    return response;
  } catch (error) {
    // 异常: 增加失败计数
    incrementFailureCount(key);
    throw error;
  }
}
```

## 完整实现

### 数据结构

```typescript
type AttemptRecord = {
  firstAtMs: number;      // 第一次失败的时间
  failureCount: number;   // 失败次数(不包括成功的请求)
};

const attempts = new Map<string, AttemptRecord>();
```

### 配置参数

```typescript
const ATTEMPT_WINDOW_MS = 5 * 60_000;  // 5 分钟窗口
const MAX_ATTEMPTS_PER_KEY = 1;        // 最多 1 次重试(总共 2 次请求)
const RETRY_DELAY_MS = 1000;           // 重试前等待 1 秒
```

### 检查重试次数

```typescript
function checkRetryLimit(key: string): void {
  const record = attempts.get(key);
  
  // 检查是否超过重试次数(只计算失败次数)
  if (record && record.failureCount > MAX_ATTEMPTS_PER_KEY) {
    throw new Error(
      `已超过最大重试次数(${MAX_ATTEMPTS_PER_KEY}),请求被阻断`
    );
  }
  
  // 如果是重试请求,等待一段时间
  if (record && record.failureCount > 0) {
    console.warn(
      `LLM 请求重试中(第 ${record.failureCount} 次失败后重试),等待 ${RETRY_DELAY_MS}ms...`
    );
    await new Promise(resolve => setTimeout(resolve, RETRY_DELAY_MS));
  }
}
```

### 更新失败计数

```typescript
function incrementFailureCount(key: string): void {
  const now = Date.now();
  const current = attempts.get(key);
  
  if (!current) {
    // 第一次失败
    attempts.set(key, { firstAtMs: now, failureCount: 1 });
  } else {
    // 增加失败计数
    attempts.set(key, { 
      firstAtMs: current.firstAtMs, 
      failureCount: current.failureCount + 1 
    });
  }
}
```

### 清理过期记录

```typescript
function pruneAttempts(): void {
  const now = Date.now();
  for (const [key, record] of attempts.entries()) {
    if (now - record.firstAtMs > ATTEMPT_WINDOW_MS) {
      attempts.delete(key);
    }
  }
}
```

## 关键决策点

### 1. 什么算"失败"?

| 情况 | 是否算失败 | 处理方式 |
|------|-----------|---------|
| HTTP 2xx | ❌ 不算 | 清除计数 |
| HTTP 4xx | ✅ 算 | 增加计数 |
| HTTP 5xx | ✅ 算 | 增加计数 |
| 网络错误 | ✅ 算 | 增加计数 |
| 超时 | ✅ 算 | 增加计数 |

### 2. 什么时候重试?

```typescript
// 只在失败次数未超限时重试
if (record.failureCount <= MAX_ATTEMPTS_PER_KEY) {
  // 等待一段时间后重试
  await delay(RETRY_DELAY_MS);
  return await executeRequest(key, request);
} else {
  // 超过限制,抛出错误
  throw new Error('已超过最大重试次数');
}
```

### 3. 什么时候清除计数?

```typescript
// 成功时立即清除
if (response.ok) {
  attempts.delete(key);
}

// 或者在时间窗口过期后清除
if (now - record.firstAtMs > ATTEMPT_WINDOW_MS) {
  attempts.delete(key);
}
```

## 测试验证

### 单元测试

```typescript
describe('重试机制', () => {
  it('成功请求不计入重试次数', async () => {
    const key = 'test-key';
    
    // 第一次成功
    await executeRequest(key, successRequest);
    expect(attempts.has(key)).toBe(false);
    
    // 第二次成功
    await executeRequest(key, successRequest);
    expect(attempts.has(key)).toBe(false);
  });
  
  it('失败请求增加重试次数', async () => {
    const key = 'test-key';
    
    // 第一次失败
    await executeRequest(key, failRequest).catch(() => {});
    expect(attempts.get(key)?.failureCount).toBe(1);
    
    // 第二次失败
    await executeRequest(key, failRequest).catch(() => {});
    expect(attempts.get(key)?.failureCount).toBe(2);
  });
  
  it('超过重试次数后抛出错误', async () => {
    const key = 'test-key';
    
    // 第一次失败
    await executeRequest(key, failRequest).catch(() => {});
    
    // 第二次失败(超过限制)
    await expect(
      executeRequest(key, failRequest)
    ).rejects.toThrow('已超过最大重试次数');
  });
  
  it('成功后清除失败计数', async () => {
    const key = 'test-key';
    
    // 第一次失败
    await executeRequest(key, failRequest).catch(() => {});
    expect(attempts.get(key)?.failureCount).toBe(1);
    
    // 第二次成功
    await executeRequest(key, successRequest);
    expect(attempts.has(key)).toBe(false);
  });
});
```

## 常见错误

### 错误 1: 不区分成功和失败

```typescript
// ❌ 错误
attempts.set(key, { count: current.count + 1 });
```

**后果**: 成功请求也被计入,很快超过限制

### 错误 2: 成功后不清除计数

```typescript
// ❌ 错误
if (response.ok) {
  // 什么都不做
}
```

**后果**: 一次失败后,即使后续都成功,仍然会被限制

### 错误 3: 重试延迟不合理

```typescript
// ❌ 错误: 延迟太短
const RETRY_DELAY_MS = 10;

// ❌ 错误: 延迟太长
const RETRY_DELAY_MS = 60_000;
```

**建议**: 1-3 秒是合理的延迟

### 错误 4: 没有时间窗口

```typescript
// ❌ 错误: 永久记录失败
const attempts = new Map<string, number>();
```

**后果**: 一次失败后永远无法重试

## 最佳实践总结

### ✅ 必须做

1. **区分成功和失败**: 只计算失败次数
2. **成功时清除计数**: 避免误判
3. **添加时间窗口**: 过期自动清理
4. **合理的重试延迟**: 1-3 秒
5. **清晰的日志输出**: 显示失败次数和重试原因

### ❌ 不要做

1. 不要把所有请求都计数
2. 不要在成功后保留失败计数
3. 不要无限重试
4. 不要立即重试(没有延迟)
5. 不要忽略时间窗口
