# AI 工具调用行为修正方法论

> **核心原则**：当 AI 自主调用工具时使用了错误的命令/参数，需要在系统层面引导 AI 的行为。

---

## 问题识别

### 典型特征

1. **AI 自主决策错误**
   - AI 调用工具时使用了错误的命令
   - AI 选择了不适合当前环境的参数
   - AI 的行为不符合平台/环境要求

2. **不是用户输入问题**
   - 用户没有明确指定错误的命令
   - 是 AI 基于训练数据或上下文做出的决策
   - 问题出现在 AI 的工具调用层面

3. **需要系统层面修正**
   - 单次纠正无效（下次还会犯同样的错误）
   - 需要在系统提示词或工具定义中引导
   - 需要持久化的行为约束

### 实战案例

**案例 1：Windows 环境下使用 Linux 命令**

- **问题**：AI 在 Windows 环境下调用 `exec` 工具时使用了 `grep` 命令
- **原因**：AI 基于训练数据中的 Linux 命令习惯，默认使用 `grep`
- **影响**：命令执行失败（`grep` 在 Windows PowerShell 中不存在）

## 解决方案

### 方案 1：在系统提示词中添加指导（推荐）⭐

**适用场景**：
- 需要全局性的行为约束
- 影响多个工具的调用
- 需要根据平台/环境动态调整

**实施步骤**：

1. **识别平台/环境**
   ```typescript
   const isWindows = runtimeInfo?.os === "win32" || process.platform === "win32";
   ```

2. **构建指导内容**
   ```typescript
   const platformGuidance = isWindows
     ? [
         "",
         "## Platform Commands (Windows)",
         "You are running on Windows. Use PowerShell commands:",
         "- File search: `Select-String -Path . -Pattern \"pattern\" -Recurse -Encoding UTF8`",
         "- List files: `Get-ChildItem -Recurse`",
         "- Read file: `Get-Content \"file.txt\" -Encoding UTF8`",
         "- **DO NOT use Linux commands**: `grep`, `find`, `cat`, `ls` are not available on Windows",
         "",
       ]
     : [];
   ```

3. **插入系统提示词**
   ```typescript
   const lines = [
     l10n.identityLine,
     "",
     l10n.toolingTitle,
     // ... 工具列表
     ...platformGuidance,  // 插入平台指导
     // ... 其他内容
   ];
   ```

**优点**：
- 全局生效，影响所有工具调用
- 可以根据环境动态调整
- AI 在每次对话开始时就能看到指导

**缺点**：
- 增加系统提示词长度
- 需要重启 Gateway 生效

### 方案 2：在工具描述中添加提示

**适用场景**：
- 只影响特定工具
- 需要针对性的使用提示
- 作为方案 1 的补充

**实施步骤**：

1. **在工具定义中添加平台提示**
   ```typescript
   const platformHint = process.platform === "win32"
     ? " On Windows, use PowerShell commands (Select-String, Get-ChildItem, Get-Content), not Linux commands (grep, find, cat)."
     : "";

   return {
     name: "exec",
     description: "Execute shell commands..." + platformHint,
     // ...
   };
   ```

**优点**：
- 修改范围小
- 直接在工具描述中提示

**缺点**：
- 只对特定工具有效
- 如果 AI 忽略描述，仍然会出错

---

### 方案 3：在工具实现中添加验证和转换

**适用场景**：
- 需要强制约束（不允许错误命令）
- 可以自动转换命令
- 作为最后一道防线

**实施步骤**：

1. **检测错误命令**
   ```typescript
   function detectInvalidCommand(command: string, platform: string): boolean {
     if (platform !== "win32") return false;
     
     // 检测 Linux 命令
     return /^(grep|find|cat|ls)\s+/i.test(command.trimStart());
   }
   ```

2. **返回错误提示**
   ```typescript
   if (detectInvalidCommand(params.command, process.platform)) {
     throw new Error(
       "Linux command not available on Windows. " +
       "Use PowerShell commands: Select-String, Get-ChildItem, Get-Content"
     );
   }
   ```

3. **或者自动转换**（可选）
   ```typescript
   function maybeConvertCommand(command: string): { command: string; warning?: string } {
     if (process.platform !== "win32") return { command };
     
     // 转换 grep 到 Select-String
     if (/^grep\s+/i.test(command)) {
       const converted = convertGrepToSelectString(command);
       return {
         command: converted,
         warning: "Converted Linux 'grep' to PowerShell 'Select-String'",
       };
     }
     
     return { command };
   }
   ```

**优点**：
- 强制约束，不允许错误命令
- 可以自动转换（如果实现了转换逻辑）

**缺点**：
- 实现复杂
- 可能无法覆盖所有情况
- 转换逻辑可能不完美

---

## 推荐策略

### 组合方案：方案 1 + 方案 2

**双层防护**：
1. **系统提示词**（主要防护）
   - 让 AI 从源头就知道应该使用什么命令
   - 全局生效

2. **工具描述**（辅助防护）
   - 在工具调用时再次提醒
   - 针对性提示

3. **工具验证**（可选，作为最后防线）
   - 检测到错误命令时返回友好的错误提示
   - 不自动转换（避免复杂性）

## 常见场景

### 场景 1：平台相关命令

**问题**：AI 使用了不适合当前平台的命令

**解决**：
- 在系统提示词中添加平台命令规范
- 明确列出可用命令和禁止命令

**示例**：
- Windows：使用 PowerShell 命令，禁止 Linux 命令
- Linux：使用 Bash 命令
- macOS：使用 Bash/Zsh 命令

### 场景 2：工具版本差异

**问题**：AI 使用了旧版本工具的语法

**解决**：
- 在工具描述中说明当前版本
- 提供正确的语法示例

**示例**：
- Docker：使用 `docker compose`（新版），不是 `docker-compose`（旧版）
- Git：使用 `git switch`（新版），不是 `git checkout`（旧版）

### 场景 3：环境特定约束

**问题**：AI 使用了在当前环境不可用的功能

**解决**：
- 在系统提示词中说明环境约束
- 提供替代方案

**示例**：
- 沙箱环境：禁止访问宿主机文件系统
- 受限环境：禁止使用某些网络功能

## 关键教训

1. **不要假设 AI 会自动适应平台**
   - AI 基于训练数据做决策，可能不适合当前环境
   - 需要明确的指导

2. **双层防护更可靠**
   - 系统提示词 + 工具描述
   - 主要防护 + 辅助防护

3. **提供具体示例**
   - 不要只说"使用 PowerShell 命令"
   - 要给出具体的命令示例

4. **明确禁止项**
   - 不要只说"应该用什么"
   - 也要说"不要用什么"

5. **支持多语言**
   - 如果系统支持多语言，指导内容也要支持
   - 确保所有语言的用户都能看到指导
