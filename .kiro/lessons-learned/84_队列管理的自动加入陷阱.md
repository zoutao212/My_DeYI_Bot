# 队列管理的自动加入陷阱

**日期**：2026-02-04  
**问题类型**：队列管理、并发控制、数据冲突  
**严重程度**：⭐⭐⭐ 高

---

## 问题描述

当系统同时支持"自动加入队列"和"工具调用创建队列任务"时，如果没有互斥机制，会导致队列中有重复或错误的任务。

---

## 典型场景

### 场景 1：自动加入与工具调用冲突

```typescript
// ❌ 错误：没有检查队列状态
if (shouldFollowup) {
  enqueueFollowupRun(queueKey, followupRun, resolvedQueue);
}

// 问题：
// 1. LLM 通过工具调用创建了 5 个队列任务
// 2. 系统又自动加入了用户消息
// 3. 队列中有 6 个任务（5 个正确 + 1 个错误）
```

### 场景 2：队列任务顺序错误

```typescript
// ❌ 错误：自动加入的任务排在前面
队列：
1. 用户消息（自动加入）❌
2. 队列任务 1（工具调用）✅
3. 队列任务 2（工具调用）✅
4. 队列任务 3（工具调用）✅

// 问题：第一个任务是用户消息，而不是队列任务
```

---

## 根本原因

1. **缺少队列状态检查**
   - 自动加入逻辑没有检查队列中是否已经有任务
   - 不知道 LLM 是否已经创建了队列任务

2. **自动加入与工具调用冲突**
   - 自动加入逻辑与工具调用逻辑是独立的
   - 没有互斥机制

3. **优先级不明确**
   - 不知道哪个任务应该先执行
   - 自动加入的任务可能排在工具调用任务前面

---

## 解决方案

### 方案 1：检查队列状态（推荐）

```typescript
// ✅ 正确：检查队列中是否已经有任务
const queueDepth = getFollowupQueueDepth(queueKey);
const hasQueuedTasks = queueDepth > 0;

if (shouldFollowup) {
  if (hasQueuedTasks) {
    // 队列中已经有任务，不要再加入用户消息
    console.log(`Queue already has ${queueDepth} tasks, skipping user message enqueue`);
  } else {
    // 队列中没有任务，加入用户消息
    enqueueFollowupRun(queueKey, followupRun, resolvedQueue);
  }
}
```

**优点**：
- 简单直接
- 避免重复加入
- 保持队列任务的正确顺序

### 方案 2：优先级设计

```typescript
// ✅ 正确：工具调用创建的任务优先级更高
const priority = {
  toolCall: 1,    // 工具调用创建的任务
  autoEnqueue: 2, // 自动加入的任务
};

// 按优先级排序队列
queue.sort((a, b) => a.priority - b.priority);
```

### 方案 3：互斥标记

```typescript
// ✅ 正确：使用标记防止重复加入
let hasEnqueuedByTool = false;

// 工具调用时设置标记
function enqueueTaskTool() {
  hasEnqueuedByTool = true;
  enqueueFollowupRun(...);
}

// 自动加入时检查标记
if (shouldFollowup && !hasEnqueuedByTool) {
  enqueueFollowupRun(...);
}
```

## 实战案例

### 案例：Hook 副作用导致队列任务 prompt 错误

**问题**：
- 用户消息："帮我生成 5 段内容..."
- LLM 调用 5 次 `enqueue_task`，创建 5 个队列任务
- 系统又自动加入了用户消息
- 队列中有 6 个任务（1 个用户消息 + 5 个队列任务）
- 第一个任务是用户消息，prompt 是 "帮我生成 5 段内容..."
- LLM 看到这个 prompt，又尝试调用 5 次 `enqueue_task`

**修复**：
```typescript
// 检查队列深度
const queueDepth = getFollowupQueueDepth(queueKey);
const hasQueuedTasks = queueDepth > 0;

if (shouldFollowup) {
  if (hasQueuedTasks) {
    console.log(`Queue already has ${queueDepth} tasks, skipping user message enqueue`);
  } else {
    enqueueFollowupRun(queueKey, followupRun, resolvedQueue);
  }
}
```

**效果**：
- 队列中只有 5 个任务（LLM 创建的）
- 第一个任务的 prompt 是 "请生成第 1 段内容..."
- LLM 不会重复调用 `enqueue_task`

## 预防措施

### 1. 设计队列管理时考虑并发

```typescript
// ✅ 正确：考虑多个来源的任务
// 1. 自动加入
// 2. 工具调用
// 3. 外部 API

// 设计互斥机制
if (hasTasksFromToolCall) {
  skipAutoEnqueue();
}
```

### 2. 添加队列状态日志

```typescript
// ✅ 正确：打印队列状态
console.log(`Queue depth: ${queueDepth}`);
console.log(`Has queued tasks: ${hasQueuedTasks}`);
console.log(`Auto enqueue: ${shouldAutoEnqueue}`);
```

### 3. 验证队列任务顺序

```typescript
// ✅ 正确：验证队列任务顺序
const firstTask = queue[0];
console.log(`First task prompt: ${firstTask.prompt}`);

// 预期：第一个任务应该是 LLM 创建的队列任务
// 而不是用户消息
```

## 关键教训

1. **队列管理要考虑并发**
   - 多个来源可能同时加入任务
   - 需要互斥机制

2. **检查队列状态**
   - 在加入任务前，检查队列中是否已经有任务
   - 避免重复加入

3. **优先级要明确**
   - 工具调用创建的任务 > 自动加入的任务
   - 确保队列任务的正确顺序

4. **日志要完整**
   - 打印队列深度
   - 打印任务来源
   - 打印任务顺序
