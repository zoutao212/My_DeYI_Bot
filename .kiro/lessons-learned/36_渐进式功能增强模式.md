# 渐进式功能增强模式

**问题类型**：如何在不破坏现有功能的前提下，逐步增强系统能力

**关键词**：渐进式开发、功能增强、向后兼容、可选参数、测试驱动

---

## 问题描述

当需要为现有系统添加新功能时，如何确保：
1. 不破坏现有功能
2. 新功能可选（向后兼容）
3. 易于测试和验证
4. 可以逐步推广

### 典型场景

1. **增强数据结构**：为现有接口添加新字段
2. **扩展功能**：为现有函数添加新能力
3. **优化性能**：在不改变接口的前提下优化实现
4. **集成新模块**：将新模块集成到现有系统

---

## 解决方案

### 核心原则

1. **可选参数**：新功能通过可选参数启用
2. **向后兼容**：不修改现有接口的必需参数
3. **渐进式测试**：先测试新功能，再测试集成
4. **文档同步**：同步更新文档和测试

### 实现模式

#### 阶段 1：扩展接口（添加可选字段）

```typescript
// 之前
interface SessionSummary {
  taskGoal: string;
  keyActions: string[];
  totalTurns: number;
}

// 之后（向后兼容）
interface SessionSummary {
  taskGoal: string;
  keyActions: string[];
  totalTurns: number;
  // 🆕 新增可选字段
  progress?: {
    completed: number;
    total: number;
    percentage: number;
  };
  nextSteps?: string[];
  keyFiles?: string[];
}
```

**关键点**：
- 使用 `?` 标记可选字段
- 不修改现有字段
- 保持向后兼容

#### 阶段 2：实现提取逻辑（独立函数）

```typescript
// 新功能独立实现
function extractProgress(messages: AgentMessage[]): ProgressInfo | undefined {
  // ... 实现
  return progress;
}

function extractNextSteps(messages: AgentMessage[]): string[] | undefined {
  // ... 实现
  return nextSteps;
}

function extractKeyFiles(messages: AgentMessage[]): string[] | undefined {
  // ... 实现
  return keyFiles;
}
```

**关键点**：
- 每个新功能独立实现
- 返回 `undefined` 表示未提取到信息
- 不依赖现有逻辑

#### 阶段 3：集成到主流程（可选调用）

```typescript
function generateSessionSummary(messages: AgentMessage[]): SessionSummary | null {
  // ... 现有逻辑
  
  // 🆕 可选调用新功能
  const progress = extractProgress(messages);
  const nextSteps = extractNextSteps(messages);
  const keyFiles = extractKeyFiles(messages);
  
  return {
    // ... 现有字段
    progress,      // 可能是 undefined
    nextSteps,     // 可能是 undefined
    keyFiles,      // 可能是 undefined
  };
}
```

**关键点**：
- 新功能可选调用
- 不影响现有逻辑
- 返回值包含新字段（可能为 undefined）

#### 阶段 4：更新使用方（可选使用）

```typescript
function formatSessionSummary(summary: SessionSummary): string {
  const parts = [
    // ... 现有格式化
  ];
  
  // 🆕 可选格式化新字段
  if (summary.progress) {
    parts.push("");
    parts.push(`**进度**：${summary.progress.completed}/${summary.progress.total} (${summary.progress.percentage}%)`);
  }
  
  if (summary.nextSteps && summary.nextSteps.length > 0) {
    parts.push("");
    parts.push("**下一步计划**：");
    parts.push(summary.nextSteps.map((s, i) => `${i + 1}. ${s}`).join("\n"));
  }
  
  if (summary.keyFiles && summary.keyFiles.length > 0) {
    parts.push("");
    parts.push("**关键文件**：");
    parts.push(summary.keyFiles.map((f) => `- ${f}`).join("\n"));
  }
  
  return parts.join("\n");
}
```

**关键点**：
- 检查字段是否存在
- 只在有数据时格式化
- 不影响现有输出

#### 阶段 5：添加测试（独立测试 + 集成测试）

```typescript
describe("新功能测试", () => {
  // 独立测试
  it("should extract progress", () => {
    const messages = [/* ... */];
    const progress = extractProgress(messages);
    expect(progress).toEqual({ completed: 3, total: 5, percentage: 60 });
  });
  
  // 集成测试
  it("should include progress in summary", () => {
    const messages = [/* ... */];
    const summary = generateSessionSummary(messages);
    expect(summary?.progress).toEqual({ completed: 3, total: 5, percentage: 60 });
  });
  
  // 向后兼容测试
  it("should work without new fields", () => {
    const messages = [/* 不包含新信息 */];
    const summary = generateSessionSummary(messages);
    expect(summary?.progress).toBeUndefined();
    expect(summary?.taskGoal).toBeDefined(); // 现有字段仍然工作
  });
});
```

**关键点**：
- 独立测试新功能
- 集成测试完整流程
- 向后兼容测试

## 最佳实践

### 1. 接口设计

**原则**：
- 新字段使用可选类型（`?`）
- 不修改现有字段的类型
- 不删除现有字段

**示例**：
```typescript
// ✅ 好的设计
interface Config {
  host: string;           // 现有必需字段
  port: number;           // 现有必需字段
  timeout?: number;       // 🆕 新增可选字段
  retries?: number;       // 🆕 新增可选字段
}

// ❌ 不好的设计
interface Config {
  host: string;
  port: number;
  timeout: number;        // ❌ 新增必需字段（破坏向后兼容）
}
```

### 2. 函数设计

**原则**：
- 新功能独立实现
- 返回 `undefined` 表示未提取到信息
- 不依赖现有逻辑

**示例**：
```typescript
// ✅ 好的设计
function extractNewFeature(data: Data): NewFeature | undefined {
  // 独立实现
  if (/* 条件 */) {
    return newFeature;
  }
  return undefined; // 明确返回 undefined
}

// ❌ 不好的设计
function extractNewFeature(data: Data): NewFeature {
  // 依赖现有逻辑
  const existing = extractExisting(data);
  return { ...existing, newField: "value" }; // ❌ 耦合
}
```

### 3. 集成策略

**原则**：
- 可选调用新功能
- 不影响现有逻辑
- 提供默认值

**示例**：
```typescript
// ✅ 好的集成
function process(data: Data): Result {
  // 现有逻辑
  const existing = processExisting(data);
  
  // 🆕 可选调用新功能
  const newFeature = extractNewFeature(data);
  
  return {
    ...existing,
    newFeature, // 可能是 undefined
  };
}

// ❌ 不好的集成
function process(data: Data): Result {
  // 强制调用新功能
  const newFeature = extractNewFeature(data);
  if (!newFeature) {
    throw new Error("New feature required"); // ❌ 破坏向后兼容
  }
  // ...
}
```

### 4. 测试策略

**原则**：
- 独立测试新功能
- 集成测试完整流程
- 向后兼容测试

**示例**：
```typescript
describe("渐进式功能增强", () => {
  // 1. 独立测试
  describe("新功能", () => {
    it("should work independently", () => {
      const result = extractNewFeature(data);
      expect(result).toBeDefined();
    });
  });
  
  // 2. 集成测试
  describe("集成", () => {
    it("should integrate with existing logic", () => {
      const result = process(data);
      expect(result.existing).toBeDefined();
      expect(result.newFeature).toBeDefined();
    });
  });
  
  // 3. 向后兼容测试
  describe("向后兼容", () => {
    it("should work without new feature", () => {
      const result = process(dataWithoutNewInfo);
      expect(result.existing).toBeDefined();
      expect(result.newFeature).toBeUndefined();
    });
  });
});
```

## 实战案例

### 案例：增强会话摘要功能

**需求**：为会话摘要添加进度、下一步计划、关键文件

**实施步骤**：

#### 步骤 1：扩展接口

```typescript
interface SessionSummary {
  taskGoal: string;
  keyActions: string[];
  keyDecisions: string[];
  blockers: string[];
  totalTurns: number;
  createdAt: number;
  // 🆕 新增可选字段
  progress?: {
    completed: number;
    total: number;
    percentage: number;
  };
  nextSteps?: string[];
  keyFiles?: string[];
}
```

#### 步骤 2：实现提取函数

```typescript
function extractProgress(messages: AgentMessage[]): ProgressInfo | undefined {
  // 独立实现
  for (const msg of messages) {
    if (msg.role === "assistant") {
      const text = extractTextFromContent(msg.content);
      const match = text.match(/(?:完成|已完成)\s*(\d+)\s*\/\s*(\d+)/);
      if (match) {
        return {
          completed: parseInt(match[1], 10),
          total: parseInt(match[2], 10),
          percentage: Math.round((parseInt(match[1], 10) / parseInt(match[2], 10)) * 100),
        };
      }
    }
  }
  return undefined;
}

// 类似地实现 extractNextSteps 和 extractKeyFiles
```

#### 步骤 3：集成到主流程

```typescript
function generateSessionSummary(messages: AgentMessage[]): SessionSummary | null {
  // ... 现有逻辑
  
  // 🆕 可选调用新功能
  const progress = extractProgress(messages);
  const nextSteps = extractNextSteps(messages);
  const keyFiles = extractKeyFiles(messages);
  
  return {
    taskGoal,
    keyActions,
    keyDecisions,
    blockers,
    totalTurns,
    createdAt,
    progress,
    nextSteps,
    keyFiles,
  };
}
```

#### 步骤 4：更新格式化

```typescript
function formatSessionSummary(summary: SessionSummary): string {
  const parts = [
    "## 会话上下文（Session Context）",
    "",
    `**任务目标**：${summary.taskGoal}`,
    `**对话轮数**：${summary.totalTurns} 轮`,
  ];
  
  // ... 现有格式化
  
  // 🆕 可选格式化新字段
  if (summary.progress) {
    parts.push("");
    parts.push(`**进度**：${summary.progress.completed}/${summary.progress.total} (${summary.progress.percentage}%)`);
  }
  
  if (summary.nextSteps && summary.nextSteps.length > 0) {
    parts.push("");
    parts.push("**下一步计划**：");
    parts.push(summary.nextSteps.map((s, i) => `${i + 1}. ${s}`).join("\n"));
  }
  
  if (summary.keyFiles && summary.keyFiles.length > 0) {
    parts.push("");
    parts.push("**关键文件**：");
    parts.push(summary.keyFiles.map((f) => `- ${f}`).join("\n"));
  }
  
  return parts.join("\n");
}
```

#### 步骤 5：添加测试

```typescript
describe("增强会话摘要", () => {
  it("should extract progress", () => {
    const messages = [
      { role: "assistant", content: [{ type: "text", text: "已完成 3/5" }] },
    ];
    const summary = generateSessionSummary(messages);
    expect(summary?.progress).toEqual({ completed: 3, total: 5, percentage: 60 });
  });
  
  it("should work without new fields", () => {
    const messages = [
      { role: "user", content: [{ type: "text", text: "任务" }] },
    ];
    const summary = generateSessionSummary(messages);
    expect(summary?.taskGoal).toBeDefined();
    expect(summary?.progress).toBeUndefined();
  });
});
```

**效果**：
- ✅ 不破坏现有功能
- ✅ 新功能可选
- ✅ 易于测试
- ✅ 向后兼容

## 常见错误

### 错误 1：新增必需字段

```typescript
// ❌ 错误：破坏向后兼容
interface Config {
  host: string;
  port: number;
  timeout: number; // 新增必需字段
}

// ✅ 正确：使用可选字段
interface Config {
  host: string;
  port: number;
  timeout?: number; // 新增可选字段
}
```

### 错误 2：强制调用新功能

```typescript
// ❌ 错误：强制调用
function process(data: Data): Result {
  const newFeature = extractNewFeature(data);
  if (!newFeature) {
    throw new Error("Required"); // 破坏向后兼容
  }
  // ...
}

// ✅ 正确：可选调用
function process(data: Data): Result {
  const newFeature = extractNewFeature(data);
  return {
    existing: processExisting(data),
    newFeature, // 可能是 undefined
  };
}
```

### 错误 3：修改现有接口

```typescript
// ❌ 错误：修改现有字段类型
interface Summary {
  taskGoal: string | TaskGoal; // 修改类型
}

// ✅ 正确：添加新字段
interface Summary {
  taskGoal: string;
  taskGoalDetails?: TaskGoal; // 新增字段
}
```

### 错误 4：缺少向后兼容测试

```typescript
// ❌ 错误：只测试新功能
describe("新功能", () => {
  it("should work", () => {
    const result = process(dataWithNewInfo);
    expect(result.newFeature).toBeDefined();
  });
});

// ✅ 正确：测试向后兼容
describe("新功能", () => {
  it("should work with new info", () => {
    const result = process(dataWithNewInfo);
    expect(result.newFeature).toBeDefined();
  });
  
  it("should work without new info", () => {
    const result = process(dataWithoutNewInfo);
    expect(result.existing).toBeDefined();
    expect(result.newFeature).toBeUndefined();
  });
});
```

## 关键教训

1. **可选参数**：新功能通过可选参数启用，不破坏现有接口
2. **独立实现**：新功能独立实现，不依赖现有逻辑
3. **渐进式测试**：独立测试 → 集成测试 → 向后兼容测试
4. **文档同步**：同步更新接口文档、使用文档、测试文档
5. **明确返回**：返回 `undefined` 表示未提取到信息，便于判断
6. **检查存在**：使用前检查字段是否存在，避免运行时错误

## 扩展模式

### 1. 配置驱动的功能开关

```typescript
interface Config {
  enableNewFeature?: boolean; // 功能开关
  newFeatureOptions?: {
    // 新功能配置
  };
}

function process(data: Data, config: Config): Result {
  const existing = processExisting(data);
  
  // 根据配置决定是否启用新功能
  if (config.enableNewFeature) {
    const newFeature = extractNewFeature(data, config.newFeatureOptions);
    return { ...existing, newFeature };
  }
  
  return existing;
}
```

### 2. 版本化的接口

```typescript
interface SummaryV1 {
  taskGoal: string;
  keyActions: string[];
}

interface SummaryV2 extends SummaryV1 {
  progress?: ProgressInfo;
  nextSteps?: string[];
}

function generateSummary(messages: AgentMessage[], version: 1 | 2 = 2): SummaryV1 | SummaryV2 {
  const v1 = generateSummaryV1(messages);
  
  if (version === 1) {
    return v1;
  }
  
  // V2 增强
  return {
    ...v1,
    progress: extractProgress(messages),
    nextSteps: extractNextSteps(messages),
  };
}
```
