# LLM 无限循环的系统提示词修复

**问题类型**：LLM 行为异常、无限循环、系统提示词设计  
**严重程度**：🔴 严重（导致系统完全不可用）  
**关键词**：LLM 无限循环、系统提示词、任务完成、重复工具调用、token 消耗

---

## 问题描述

LLM 陷入无限循环，重复执行相同的工具调用，导致：
- Token 消耗暴增（从 7695 → 95942）
- 系统完全不可用
- 用户无法得到回复

**典型现象**：
```
seq=1: Get-Date, readFile, writeFile
seq=2: Get-Date, readFile, writeFile  (完全相同)
seq=3: Get-Date, readFile, writeFile  (完全相同)
...
seq=23: Get-Date, readFile, writeFile (完全相同)
```

---

## 根本原因

### 1. 系统提示词缺少"完成"指导

系统提示词没有明确告诉 LLM：
- ✅ 什么时候任务完成
- ✅ 如何结束对话
- ✅ 如何避免重复操作

### 2. LLM 没有"完成"的概念

LLM 可能认为：
- 工具调用成功 ≠ 任务完成
- 需要继续执行更多操作
- 不知道何时停止

### 3. 系统提示词过长导致 LLM 迷失

当系统提示词包含大量历史消息和冗余信息时，LLM 可能：
- 忘记任务目标
- 陷入无限循环
- 无法正确判断任务是否完成

---

## 修复方案

### 方案 1：在系统提示词中添加"任务完成原则"

**位置**：`src/agents/system-prompt.l10n.zh.ts`

```typescript
toolCallCompletionNote:
  "**任务完成原则**:当你完成用户请求的任务后，必须立即停止工具调用并回复用户。不要重复执行相同的工具调用。如果工具调用已经成功（返回了结果），不要再次调用相同的工具。",
```

**关键点**：
1. 明确告知 LLM 何时任务完成
2. 禁止重复执行相同的工具调用
3. 强调"工具调用成功 = 任务完成"

### 方案 2：限制历史消息数量

**位置**：`src/agents/pi-embedded-runner/history.ts`

```typescript
// 限制历史消息数量，避免系统提示词过长
const historyLimit = 20; // 保留最近 20 条消息
const recentMessages = allMessages.slice(-historyLimit);
```

### 方案 3：使用精简版系统提示词

**位置**：`src/agents/system-prompt.l10n.minimal.zh.ts`

```typescript
// 精简版系统提示词，减少 token 消耗
toolCallCompletionNote: "**任务完成原则**：完成任务后立即停止工具调用并回复用户。不要重复相同的工具调用。",
```

## 关键教训

### 1. 系统提示词必须明确"完成"条件

LLM 不会自动知道任务何时完成，必须在系统提示词中明确告知：
- 什么时候任务完成
- 如何结束对话
- 如何避免重复操作

### 2. 工具调用成功 ≠ 任务完成（对 LLM 来说）

LLM 可能认为：
- 工具调用成功只是一个步骤
- 需要继续执行更多操作
- 不知道何时停止

**解决方案**：在系统提示词中明确说明"如果工具调用已经成功，不要再次调用相同的工具"。

### 3. 系统提示词过长会导致 LLM 迷失

当系统提示词包含大量历史消息和冗余信息时，LLM 可能：
- 忘记任务目标
- 陷入无限循环
- 无法正确判断任务是否完成

**解决方案**：
- 限制历史消息数量
- 移除冗余信息
- 使用精简版系统提示词

### 4. 必须在系统提示词中明确禁止重复操作

LLM 可能会：
- 重复执行相同的工具调用
- 认为需要多次验证
- 不知道一次成功就够了

**解决方案**：在系统提示词中明确说明"不要重复执行相同的工具调用"。

### 5. 系统提示词的措辞非常重要

不同的措辞会导致 LLM 有不同的理解：
- ❌ "可以停止工具调用" → LLM 可能认为是可选的
- ✅ "必须立即停止工具调用" → LLM 知道这是强制的
