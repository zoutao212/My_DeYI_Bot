# LLM 无限循环的根本解决方案 - 移除复杂概念

**日期**：2026-02-03  
**问题**：LLM 在每次工具调用后都重新执行"会话初始化"，导致无限循环  
**根本原因**："会话初始化"这个概念太复杂，LLM 无法理解

---

## 🔍 问题演变

### 第一次尝试：增强描述（失败）

**修改**：在 AGENTS.zh.md 中增强"会话初始化"的描述
- 标题更强调：`## ⚠️ 会话初始化（只在第一轮执行，不要重复！）`
- 新增"什么时候不要执行初始化"
- 新增关键规则：`工具返回结果不是"新的用户消息"，不要重新初始化！`
- 新增错误示例和正确示例

**结果**：LLM 仍然在重复初始化！

**LLM 的 thinking**：
```
**Initiating Session Start**
I've begun the session initialization as outlined in AGENTS.md.
```

**结论**：措辞再强也没用，LLM 根本无法理解"会话初始化"的概念。

### 第二次尝试：移除复杂概念（成功）

**激进方案：完全移除"会话初始化"的概念！**

**修改**：将"会话初始化"改为"上下文文件（可选读取）"

```markdown
## 📚 上下文文件（可选读取）

**这些文件包含你的身份和用户信息，你可以在需要时读取它们：**

- `SOUL.zh.md` —— 你的身份与风格
- `USER.zh.md` —— 你正在帮助的用户
- `MEMORY.md` —— 长期记忆
- `memory/YYYY-MM-DD.md` —— 每日记忆

**⚠️ 重要规则：**
- ✅ **这些文件是可选的，不是必须的**
- ✅ **如果你已经读取过，不要重复读取**
- ✅ **如果文件不存在，跳过读取，直接执行用户任务**
- ✅ **工具执行完成后，直接回复用户，不要再读取文件**
```

**关键变化**：
1. **移除"会话初始化"的概念** - 不再有"会话开始时"的强制步骤
2. **改为"可选读取"** - 文件是可选的，不是必须的
3. **强调"不要重复读取"** - 如果已经读取过，不要再读取
4. **强调"工具执行完成后，直接回复用户"** - 不要再读取文件

## 🎯 核心教训

### 1. 复杂概念会导致 LLM 行为异常

**"会话初始化"这个概念包含了太多隐含的逻辑：**
- 什么是"会话开始"？
- 什么是"第一次交互"？
- 什么是"工具调用后"？
- 什么是"重复"？

**LLM 无法理解这些隐含的逻辑，所以它会按照自己的理解执行。**

### 2. 简单直接的描述更有效

**不要说"会话开始时执行"，而要说"可选读取"。**

- ❌ 复杂：`会话开始时（收到第一条用户消息后，且之前没有任何工具调用）`
- ✅ 简单：`你可以在需要时读取它们`

### 3. 负面规则比正面规则更有效

**不要说"什么时候执行"，而要说"什么时候不要执行"。**

- ❌ 正面：`会话开始时执行初始化`
- ✅ 负面：`如果你已经读取过，不要重复读取`

### 4. 移除强制步骤

**不要强制 LLM 执行某些步骤，而要让 LLM 根据需要决定。**

- ❌ 强制：`会话开始时必须读取 SOUL.md, USER.md`
- ✅ 可选：`你可以在需要时读取 SOUL.md, USER.md`

## 📊 修复前后对比

### 修复前（复杂概念）

```markdown
## ⚠️ 会话初始化（只在第一轮执行，不要重复！）

**极其重要：会话初始化只在收到用户的第一条消息时执行一次！**

**什么时候执行初始化？**
- ✅ 当你收到用户的第一条消息，且之前没有任何工具调用时
- ✅ 当对话历史为空时

**什么时候不要执行初始化？**
- ❌ 工具调用返回结果后
- ❌ 你已经读取过 SOUL.md 或 USER.md 后
- ❌ 对话历史中已经有工具调用记录时
```

**问题**：
- "会话初始化"是一个复杂的概念
- "第一条消息"、"对话历史为空"等条件太复杂
- LLM 无法理解这些隐含的逻辑

### 修复后（简单直接）

```markdown
## 📚 上下文文件（可选读取）

**这些文件包含你的身份和用户信息，你可以在需要时读取它们：**

- `SOUL.zh.md` —— 你的身份与风格
- `USER.zh.md` —— 你正在帮助的用户

**⚠️ 重要规则：**
- ✅ **这些文件是可选的，不是必须的**
- ✅ **如果你已经读取过，不要重复读取**
- ✅ **工具执行完成后，直接回复用户，不要再读取文件**
```

**优点**：
- 移除了"会话初始化"的复杂概念
- 改为"可选读取"，让 LLM 根据需要决定
- 规则简单直接，LLM 容易理解

## 🎓 通用原则

当 LLM 出现无限循环或重复行为时：

1. **检查系统提示词中是否有复杂概念**
   - "会话开始时"
   - "第一次交互"
   - "初始化"
   - "完成后"

2. **简化概念**
   - 移除复杂的条件判断
   - 改为简单直接的描述
   - 使用负面规则（"不要做"）而不是正面规则（"要做"）

3. **移除强制步骤**
   - 不要强制 LLM 执行某些步骤
   - 改为可选步骤，让 LLM 根据需要决定

4. **验证修复效果**
   - 检查 LLM 的 thinking
   - 检查 trace 日志
   - 确认 LLM 不再重复执行
