# 队列系统调试方法论

**问题类型**：异步任务队列系统 - 任务入队但不执行

**适用场景**：
- 队列系统已实现，但任务没有被执行
- 任务只执行一次或有限次数，然后停止
- 队列中有任务，但没有被触发

---

## 核心原则

**队列系统 = 入队 + 触发 + 出队 + 执行**

任何一个环节出问题，都会导致任务无法执行。

---

## 调试流程

### 步骤 1：追踪数据流

**目标**：确认任务是否真的入队了

**方法**：
1. 找到入队函数（如 `enqueueFollowupRun`）
2. 检查调用位置
3. 添加日志验证

**示例**：
```typescript
export function enqueueFollowupRun(key: string, run: FollowupRun): boolean {
  console.log(`[DEBUG] Enqueuing task for key: ${key}`);
  console.log(`[DEBUG] Task prompt: ${run.prompt.slice(0, 50)}...`);
  
  const queue = getFollowupQueue(key);
  queue.items.push(run);
  
  console.log(`[DEBUG] Queue depth after enqueue: ${queue.items.length}`);
  return true;
}
```

**验证**：
```powershell
# 查看入队日志
Select-String -Path "runtimelog/*.jsonl" -Pattern "Enqueuing task" -Context 2,2
```

### 步骤 2：检查触发点

**目标**：确认队列是否在正确的时机被触发

**常见触发点**：
- Agent 执行完成后
- 任务入队后
- 定时器触发
- 事件监听器触发

**方法**：
1. 找到触发函数（如 `scheduleFollowupDrain`）
2. 检查所有调用位置
3. 确认是否覆盖所有执行路径

**示例**：
```typescript
// ❌ 错误：只在一个地方触发
function agentRunner() {
  // ... 执行 agent
  finalizeWithFollowup(result, queueKey, runFollowup);  // 只在这里触发
}

function followupRunner() {
  // ... 执行队列任务
  // ❌ 没有触发队列！任务执行完成后，队列不会继续
}
```

**正确做法**：
```typescript
// ✅ 正确：在所有执行路径都触发
function agentRunner() {
  // ... 执行 agent
  finalizeWithFollowup(result, queueKey, runFollowup);
}

function followupRunner() {
  // ... 执行队列任务
  finalizeWithFollowup(undefined, queueKey, runFollowup);  // ✅ 触发队列
}
```

### 步骤 3：验证递归逻辑

**目标**：确认队列执行完成后是否会继续检查

**方法**：
1. 检查队列执行函数的 `finally` 块
2. 确认是否有递归调用
3. 确认递归条件是否正确

**示例**：
```typescript
export function scheduleFollowupDrain(
  key: string,
  runFollowup: (run: FollowupRun) => Promise<void>,
): void {
  const queue = FOLLOWUP_QUEUES.get(key);
  if (!queue || queue.draining) return;
  
  queue.draining = true;
  void (async () => {
    try {
      while (queue.items.length > 0) {
        const next = queue.items.shift();
        if (!next) break;
        await runFollowup(next);
      }
    } finally {
      queue.draining = false;
      
      // ✅ 关键：检查是否还有任务
      if (queue.items.length > 0) {
        scheduleFollowupDrain(key, runFollowup);  // 递归调用
      } else {
        FOLLOWUP_QUEUES.delete(key);
      }
    }
  })();
}
```

### 步骤 4：日志追踪

**目标**：通过日志追踪队列状态变化

**关键日志点**：
1. 任务入队
2. 队列触发
3. 任务执行开始
4. 任务执行完成
5. 队列检查
6. 递归调用

**示例**：
```typescript
export function scheduleFollowupDrain(
  key: string,
  runFollowup: (run: FollowupRun) => Promise<void>,
): void {
  const queue = FOLLOWUP_QUEUES.get(key);
  
  console.log(`[QUEUE] scheduleFollowupDrain called for key: ${key}`);
  console.log(`[QUEUE] Queue exists: ${!!queue}`);
  console.log(`[QUEUE] Queue draining: ${queue?.draining}`);
  console.log(`[QUEUE] Queue items: ${queue?.items.length ?? 0}`);
  
  if (!queue || queue.draining) {
    console.log(`[QUEUE] Skipping drain (no queue or already draining)`);
    return;
  }
  
  queue.draining = true;
  void (async () => {
    try {
      while (queue.items.length > 0) {
        console.log(`[QUEUE] Processing item, remaining: ${queue.items.length}`);
        
        const next = queue.items.shift();
        if (!next) break;
        
        console.log(`[QUEUE] Executing task: ${next.prompt.slice(0, 50)}...`);
        await runFollowup(next);
        console.log(`[QUEUE] Task completed`);
      }
    } finally {
      queue.draining = false;
      
      console.log(`[QUEUE] Drain finished, items: ${queue.items.length}`);
      
      if (queue.items.length > 0) {
        console.log(`[QUEUE] Recursing to process remaining items`);
        scheduleFollowupDrain(key, runFollowup);
      } else {
        console.log(`[QUEUE] Queue empty, deleting`);
        FOLLOWUP_QUEUES.delete(key);
      }
    }
  })();
}
```

## 常见问题模式

### 问题 1：任务入队但不触发

**症状**：
- 队列中有任务
- 但 `scheduleFollowupDrain` 没有被调用

**原因**：
- 触发点缺失
- 触发条件不满足

**解决**：
- 在所有执行路径添加触发点
- 检查触发条件

### 问题 2：任务只执行一次

**症状**：
- 第一个任务执行成功
- 后续任务不执行

**原因**：
- 任务执行完成后没有触发队列
- 递归调用缺失

**解决**：
- 在任务执行完成后添加触发点
- 确保递归逻辑正确

### 问题 3：任务执行有限次数

**症状**：
- 前几个任务执行成功
- 之后停止

**原因**：
- 某个执行路径没有触发队列
- 递归条件错误

**解决**：
- 检查所有执行路径
- 修正递归条件

## 验证方法

### 方法 1：日志分析

```powershell
# 查看队列相关日志
$trace = Get-Content "runtimelog/trace__*.jsonl" -Tail 1000 -Encoding UTF8 | ConvertFrom-Json

# 查看队列事件
$queueEvents = $trace | Where-Object { 
  $_.event -like "*queue*" -or 
  $_.message -like "*QUEUE*"
}

$queueEvents | Select-Object -Last 20 | ConvertTo-Json -Depth 5
```

### 方法 2：断点调试

在关键位置添加断点：
1. 入队函数
2. 触发函数
3. 执行函数
4. 递归调用

### 方法 3：单元测试

```typescript
describe("Queue System", () => {
  it("should execute all tasks in queue", async () => {
    const key = "test-queue";
    const executed: string[] = [];
    
    // 入队 3 个任务
    enqueueFollowupRun(key, createTask("task1"));
    enqueueFollowupRun(key, createTask("task2"));
    enqueueFollowupRun(key, createTask("task3"));
    
    // 触发队列
    scheduleFollowupDrain(key, async (run) => {
      executed.push(run.prompt);
    });
    
    // 等待执行完成
    await waitForQueueEmpty(key);
    
    // 验证所有任务都执行了
    expect(executed).toEqual(["task1", "task2", "task3"]);
  });
});
```

## 实战案例：Clawdbot 连续任务

**问题**：连续任务最多只能回复 2 次

**分析**：
1. **入队**：✅ `enqueueFollowupRun` 正常工作
2. **触发**：⚠️ 只在 `agent-runner.ts` 中触发
3. **执行**：✅ `followup-runner.ts` 正常执行
4. **递归**：❌ `followup-runner.ts` 执行完成后没有触发队列

**解决**：
在 `followup-runner.ts` 中添加队列触发：
```typescript
// 任务执行完成后
await sendFollowupPayloads(finalPayloads, queued);

// 🆕 触发队列执行下一个任务
if (queued.run.sessionKey) {
  finalizeWithFollowup(undefined, queued.run.sessionKey, runFollowupTurn);
}
```

## 关键教训

1. **队列系统需要完整的触发机制**
   - 不能只在一个地方触发
   - 所有执行路径都要触发

2. **递归调用是关键**
   - 任务执行完成后必须检查队列
   - 直到队列为空才停止

3. **日志是调试的最佳工具**
   - 添加详细的日志
   - 追踪队列状态变化

4. **验证优先于实施**
   - 先确认问题
   - 再实施修复
