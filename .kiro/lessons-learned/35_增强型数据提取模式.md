# 增强型数据提取模式

**问题类型**：从非结构化文本中提取结构化信息

**关键词**：数据提取、正则表达式、模式匹配、信息提取、文本解析

---

## 问题描述

当需要从 LLM 生成的非结构化文本中提取结构化信息时（如进度、计划、文件列表等），如何设计一个可扩展、可维护的提取系统？

### 典型场景

1. **进度提取**：从 "已完成 3/5" 或 "进度 60%" 中提取进度信息
2. **计划提取**：从 "下一步：修复错误" 中提取下一步计划
3. **文件提取**：从工具调用参数中提取文件路径
4. **关键词提取**：从文本中提取关键决策、问题等

---

## 解决方案

### 核心原则

1. **单一职责**：每个提取函数只负责一种信息类型
2. **多模式支持**：支持多种表达方式（如分数、百分比）
3. **限制数量**：避免信息过载，限制提取数量
4. **明确返回**：返回 `undefined` 而不是空数组，便于判断

### 实现模式

#### 1. 定义提取函数

```typescript
/**
 * 提取进度信息
 * 支持模式：
 * - "完成 3/5" → { completed: 3, total: 5, percentage: 60 }
 * - "进度 60%" → { completed: 60, total: 100, percentage: 60 }
 */
function extractProgress(messages: AgentMessage[]): ProgressInfo | undefined {
  for (const msg of messages) {
    if (msg.role === "assistant") {
      const text = extractTextFromContent(msg.content);
      
      // Pattern 1: 分数格式
      const fractionMatch = text.match(/(?:完成|已完成)\s*(\d+)\s*\/\s*(\d+)/);
      if (fractionMatch) {
        const completed = parseInt(fractionMatch[1], 10);
        const total = parseInt(fractionMatch[2], 10);
        return {
          completed,
          total,
          percentage: Math.round((completed / total) * 100),
        };
      }
      
      // Pattern 2: 百分比格式
      const percentMatch = text.match(/(?:已完成|进度)\s*(\d+)%/);
      if (percentMatch) {
        const percentage = parseInt(percentMatch[1], 10);
        return {
          completed: percentage,
          total: 100,
          percentage,
        };
      }
    }
  }
  
  return undefined; // 明确返回 undefined
}
```

#### 2. 定义类型

```typescript
interface ProgressInfo {
  completed: number;
  total: number;
  percentage: number;
}

interface ExtractedData {
  progress?: ProgressInfo;
  nextSteps?: string[];
  keyFiles?: string[];
}
```

#### 3. 组合提取

```typescript
function extractAllData(messages: AgentMessage[]): ExtractedData {
  return {
    progress: extractProgress(messages),
    nextSteps: extractNextSteps(messages),
    keyFiles: extractKeyFiles(messages),
  };
}
```

## 最佳实践

### 1. 正则表达式设计

**原则**：
- 使用非捕获组 `(?:...)` 提高性能
- 使用 `\s*` 允许空格
- 使用 `\d+` 匹配数字
- 使用 `|` 支持多种关键词

**示例**：
```typescript
// ✅ 好的正则表达式
const match = text.match(/(?:完成|已完成)\s*(\d+)\s*\/\s*(\d+)/);

// ❌ 不好的正则表达式（过于严格）
const match = text.match(/完成(\d+)\/(\d+)/);
```

### 2. 限制提取数量

**原则**：
- 避免信息过载
- 只保留最重要的信息
- 使用 `slice()` 限制数量

**示例**：
```typescript
// ✅ 限制数量
return nextSteps.slice(0, 5); // 最多 5 条

// ❌ 不限制数量
return nextSteps; // 可能有 100+ 条
```

### 3. 去重

**原则**：
- 使用 `Set` 去重
- 转换回数组后再限制数量

**示例**：
```typescript
// ✅ 去重
return [...new Set(keyFiles)].slice(0, 10);

// ❌ 不去重
return keyFiles.slice(0, 10); // 可能有重复
```

### 4. 返回值设计

**原则**：
- 返回 `undefined` 而不是空数组
- 便于判断是否提取到信息
- 减少不必要的字段

**示例**：
```typescript
// ✅ 返回 undefined
return keyFiles.length > 0 ? keyFiles : undefined;

// ❌ 返回空数组
return keyFiles; // 即使为空也返回 []
```

## 扩展模式

### 1. 从工具调用中提取

```typescript
function extractKeyFiles(messages: AgentMessage[]): string[] | undefined {
  const keyFiles: string[] = [];
  
  for (const msg of messages) {
    if (msg.role === "assistant") {
      const assistantMsg = msg as { 
        role: "assistant"; 
        tool_calls?: Array<{ 
          function: { 
            name: string; 
            arguments: string 
          } 
        }> 
      };
      
      if (assistantMsg.tool_calls) {
        for (const call of assistantMsg.tool_calls) {
          if (["read", "write", "edit"].includes(call.function.name)) {
            try {
              const args = JSON.parse(call.function.arguments);
              if (args.path) {
                keyFiles.push(args.path);
              }
            } catch {
              // Ignore parse errors
            }
          }
        }
      }
    }
  }
  
  return keyFiles.length > 0 ? [...new Set(keyFiles)].slice(0, 10) : undefined;
}
```

### 2. 从关键词中提取

```typescript
function extractNextSteps(messages: AgentMessage[]): string[] | undefined {
  const nextSteps: string[] = [];
  const nextStepKeywords = ["下一步", "接下来", "然后", "之后"];
  
  for (const msg of messages) {
    if (msg.role === "assistant") {
      const text = extractTextFromContent(msg.content);
      const lines = text.split("\n");
      
      for (const line of lines) {
        if (nextStepKeywords.some((kw) => line.includes(kw))) {
          const step = line.replace(/^[*\-\d.]+\s*/, "").trim();
          if (step.length > 5 && step.length < 100) {
            nextSteps.push(step);
          }
        }
      }
    }
  }
  
  return nextSteps.length > 0 ? nextSteps.slice(0, 5) : undefined;
}
```

## 测试策略

describe("extractProgress", () => {

## 常见错误

### 错误 1：正则表达式过于严格

```typescript
// ❌ 错误：不允许空格
const match = text.match(/完成(\d+)\/(\d+)/);

// ✅ 正确：允许空格
const match = text.match(/(?:完成|已完成)\s*(\d+)\s*\/\s*(\d+)/);
```

### 错误 2：不限制提取数量

```typescript
// ❌ 错误：可能提取 100+ 条
return nextSteps;

// ✅ 正确：限制到 5 条
return nextSteps.slice(0, 5);
```

### 错误 3：返回空数组而不是 undefined

```typescript
// ❌ 错误：即使为空也返回 []
return keyFiles;

// ✅ 正确：返回 undefined
return keyFiles.length > 0 ? keyFiles : undefined;
```

### 错误 4：不去重

```typescript
// ❌ 错误：可能有重复
return keyFiles.slice(0, 10);

// ✅ 正确：去重后再限制
return [...new Set(keyFiles)].slice(0, 10);
```

## 关键教训

1. **单一职责**：每个提取函数只负责一种信息类型
2. **多模式支持**：支持多种表达方式，提高鲁棒性
3. **限制数量**：避免信息过载，只保留最重要的信息
4. **明确返回**：返回 `undefined` 而不是空数组，便于判断
5. **去重**：使用 `Set` 去重，避免重复信息
