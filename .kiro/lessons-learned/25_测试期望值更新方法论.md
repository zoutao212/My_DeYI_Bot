# 测试期望值更新方法论

> **触发场景**：修改实现后，测试失败，需要更新期望值

---

## 问题识别

### 典型场景

1. **行为改变**：修改了函数的返回值或副作用
2. **新增功能**：添加了新的字段或属性
3. **降级策略**：从一种实现切换到另一种实现

### 判断标准

**何时应该更新期望值？**
- ✅ 新行为是正确的，符合需求
- ✅ 旧期望值已经过时
- ✅ 测试逻辑仍然有效

**何时不应该更新期望值？**
- ❌ 新行为是错误的，应该修复代码
- ❌ 测试逻辑本身有问题
- ❌ 只是为了让测试通过而修改

---

## 标准更新流程

### 第一步：运行测试，查看失败信息

**运行测试**：
```powershell
pnpm test <test-file> --run
```

**分析失败信息**：
- 哪个测试失败了？
- 期望值是什么？
- 实际值是什么？
- 差异在哪里？

**示例**：
```
FAIL  src/agents/tools/memory-tool.does-not-crash-on-errors.test.ts
  > does not throw when memory_search fails

  AssertionError: expected { results: [], …(3) } to deeply equal { results: [], disabled: true, …(1) }

  - Expected
  + Received

  {
  -   "disabled": true,
  -   "error": "openai embeddings failed: 429 insufficient_quota",
  +   "fallback": true,
  +   "provider": "keyword",
      "results": [],
  +   "warning": "Using keyword search as fallback",
  }
```

**分析**：
- 期望：`disabled: true, error: "..."`
- 实际：`fallback: true, provider: "keyword", warning: "..."`
- 原因：实现从"直接失败"改为"降级到关键词搜索"

### 第二步：验证新行为是否正确

**检查代码逻辑**：
- 新行为是否符合需求？
- 新行为是否更好？
- 新行为是否有副作用？

**示例**：
```typescript
// 旧实现：直接返回 disabled
if (!manager) {
  return jsonResult({ results: [], disabled: true, error });
}

// 新实现：降级到关键词搜索
if (!manager) {
  // 降级到关键词搜索
  const keywordResults = await keywordSearch({ ... });
  return jsonResult({
    results: keywordResults,
    provider: "keyword",
    fallback: true,
    warning: "Using keyword search as fallback"
  });
}
```

**判断**：
- ✅ 新行为更好（降级比完全失效好）
- ✅ 符合需求（提供降级方案）
- ✅ 没有副作用

### 第三步：更新测试期望值

**修改测试代码**：
```typescript
// ❌ 旧期望值
expect(result.details).toEqual({
  results: [],
  disabled: true,
  error: "openai embeddings failed: 429 insufficient_quota",
});

// ✅ 新期望值
expect(result.details).toMatchObject({
  results: [],
  provider: "keyword",
  fallback: true,
});
expect(result.details).toHaveProperty("warning");
```

**添加注释说明**：
```typescript
it("does not throw when memory_search fails (e.g. embeddings 429)", async () => {
  const cfg = { agents: { list: [{ id: "main", default: true }] } };
  const tool = createMemorySearchTool({ config: cfg });
  expect(tool).not.toBeNull();
  if (!tool) throw new Error("tool missing");

  const result = await tool.execute("call_1", { query: "hello" });
  
  // 现在会降级到关键词搜索，而不是直接返回 disabled
  expect(result.details).toMatchObject({
    results: [],
    provider: "keyword",
    fallback: true,
  });
  expect(result.details).toHaveProperty("warning");
});
```

### 第四步：重新运行测试验证

**运行测试**：
```powershell
pnpm test <test-file> --run
```

**检查结果**：
- ✅ 测试通过
- ✅ 没有新的失败
- ✅ 逻辑正确

## 实战案例

### 案例 1：行为改变（降级策略）

**场景**：记忆搜索从"直接失败"改为"降级到关键词搜索"

**旧测试**：
```typescript
it("does not throw when memory_search fails", async () => {
  const result = await tool.execute("call_1", { query: "hello" });
  expect(result.details).toEqual({
    results: [],
    disabled: true,
    error: "openai embeddings failed: 429 insufficient_quota",
  });
});
```

**失败信息**：
```
- Expected
+ Received

{
-   "disabled": true,
-   "error": "openai embeddings failed: 429 insufficient_quota",
+   "fallback": true,
+   "provider": "keyword",
    "results": [],
+   "warning": "Using keyword search as fallback",
}
```

**新测试**：
```typescript
it("does not throw when memory_search fails (e.g. embeddings 429)", async () => {
  const result = await tool.execute("call_1", { query: "hello" });
  
  // 现在会降级到关键词搜索，而不是直接返回 disabled
  expect(result.details).toMatchObject({
    results: [],
    provider: "keyword",
    fallback: true,
  });
  expect(result.details).toHaveProperty("warning");
});
```

### 案例 2：新增功能（保留任务目标）

**场景**：`limitHistoryTurns` 新增保留任务目标功能

**旧测试**：
```typescript
it("limits to 2 user turns", () => {
  const messages = [
    { role: "user", content: [{ type: "text", text: "task" }], timestamp: 1 },
    { role: "assistant", content: [{ type: "text", text: "ok" }], timestamp: 2 },
    { role: "user", content: [{ type: "text", text: "q1" }], timestamp: 3 },
    { role: "assistant", content: [{ type: "text", text: "a1" }], timestamp: 4 },
    { role: "user", content: [{ type: "text", text: "q2" }], timestamp: 5 },
  ];
  
  const result = limitHistoryTurns(messages, 2);
  
  expect(result).toHaveLength(3);  // 2 user + 1 assistant
  expect(result[0].role).toBe("user");
  expect(result[0].content[0].text).toBe("q1");
});
```

**失败信息**：
```
- Expected length: 3
+ Received length: 4

Received array:
[
  { role: "user", content: [{ type: "text", text: "[任务目标 - 保留自对话开始]\n" }, ...], timestamp: 1 },
  { role: "user", content: [{ type: "text", text: "q1" }], timestamp: 3 },
  { role: "assistant", content: [{ type: "text", text: "a1" }], timestamp: 4 },
  { role: "user", content: [{ type: "text", text: "q2" }], timestamp: 5 },
]
```

**新测试**：
```typescript
it("limits to 2 user turns and preserves task goal", () => {
  const messages = [
    { role: "user", content: [{ type: "text", text: "task" }], timestamp: 1 },
    { role: "assistant", content: [{ type: "text", text: "ok" }], timestamp: 2 },
    { role: "user", content: [{ type: "text", text: "q1" }], timestamp: 3 },
    { role: "assistant", content: [{ type: "text", text: "a1" }], timestamp: 4 },
    { role: "user", content: [{ type: "text", text: "q2" }], timestamp: 5 },
  ];
  
  const result = limitHistoryTurns(messages, 2);
  
  // 现在会保留任务目标，所以是 4 条消息（任务目标 + 2 user + 1 assistant）
  expect(result).toHaveLength(4);
  expect(result[0].role).toBe("user");
  expect(result[0].content[0].text).toContain("[任务目标 - 保留自对话开始]");
  expect(result[1].role).toBe("user");
  expect(result[1].content[0].text).toBe("q1");
});
```

## 检查清单

更新测试期望值时，按以下顺序检查：

- [ ] **运行测试**：查看失败信息
- [ ] **分析差异**：理解期望值和实际值的差异
- [ ] **验证新行为**：确认新行为是正确的
- [ ] **更新期望值**：修改测试代码
- [ ] **添加注释**：说明为什么改变
- [ ] **重新运行测试**：验证测试通过
- [ ] **检查其他测试**：确保没有影响其他测试

## 常见错误模式

### 错误 1：不分析就直接更新

```typescript
// ❌ 错误：看到测试失败就直接改期望值
expect(result).toEqual(actualValue);  // 盲目复制实际值

// ✅ 正确：先分析新行为是否正确
// 1. 查看失败信息
// 2. 理解新行为
// 3. 验证新行为是否正确
// 4. 更新期望值并添加注释
```

### 错误 2：只改期望值，不改测试逻辑

```typescript
// ❌ 错误：只改期望值，测试逻辑过时
it("returns disabled when search fails", async () => {
  const result = await tool.execute("call_1", { query: "hello" });
  
  // 期望值更新了，但测试名称和逻辑还是旧的
  expect(result.details).toMatchObject({
    fallback: true,
    provider: "keyword",
  });
});

// ✅ 正确：同时更新测试名称和逻辑
it("falls back to keyword search when embedding search fails", async () => {
  const result = await tool.execute("call_1", { query: "hello" });
  
  // 测试降级行为
  expect(result.details).toMatchObject({
    fallback: true,
    provider: "keyword",
  });
  expect(result.details).toHaveProperty("warning");
});
```

### 错误 3：不添加注释说明

```typescript
// ❌ 错误：没有注释说明为什么改变
expect(result).toHaveLength(4);

// ✅ 正确：添加注释说明
// 现在会保留任务目标，所以是 4 条消息（任务目标 + 2 user + 1 assistant）
expect(result).toHaveLength(4);
```

## 使用技巧

### 技巧 1：使用 toMatchObject 而不是 toEqual

**原因**：`toMatchObject` 只检查指定的字段，更灵活

```typescript
// ❌ 不够灵活：必须完全匹配
expect(result.details).toEqual({
  results: [],
  provider: "keyword",
  fallback: true,
  warning: "Using keyword search as fallback",
});

// ✅ 更灵活：只检查关键字段
expect(result.details).toMatchObject({
  results: [],
  provider: "keyword",
  fallback: true,
});
expect(result.details).toHaveProperty("warning");
```

### 技巧 2：分步验证

**原因**：更容易定位问题

```typescript
// ❌ 一次性验证：失败时不知道哪里错了
expect(result).toEqual({
  length: 4,
  firstRole: "user",
  firstText: "[任务目标 - 保留自对话开始]",
});

// ✅ 分步验证：失败时容易定位
expect(result).toHaveLength(4);
expect(result[0].role).toBe("user");
expect(result[0].content[0].text).toContain("[任务目标 - 保留自对话开始]");
```

### 技巧 3：保留旧测试作为参考

**原因**：方便回顾历史行为

```typescript
// 保留旧测试（注释掉）作为参考
// it("returns disabled when search fails (old behavior)", async () => {
//   const result = await tool.execute("call_1", { query: "hello" });
//   expect(result.details).toEqual({
//     results: [],
//     disabled: true,
//     error: "openai embeddings failed: 429 insufficient_quota",
//   });
// });

// 新测试
it("falls back to keyword search when embedding search fails", async () => {
  const result = await tool.execute("call_1", { query: "hello" });
  expect(result.details).toMatchObject({
    results: [],
    provider: "keyword",
    fallback: true,
  });
});
```

## 关键教训

1. **先分析，再更新** - 不要盲目复制实际值
2. **验证新行为** - 确保新行为是正确的
3. **添加注释** - 说明为什么改变
4. **更新测试逻辑** - 不只是期望值，测试名称和逻辑也要更新
5. **分步验证** - 更容易定位问题
6. **使用灵活的断言** - `toMatchObject` 比 `toEqual` 更灵活
