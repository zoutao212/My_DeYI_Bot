# 连续任务队列触发机制修复

**日期**：2026-02-04  
**问题**：连续任务最多只能回复 2 次就中断  
**根本原因**：队列系统已实现，但触发机制不完整

---

## 问题现象

### 用户报告

用户发送需要多次回复的任务时：
- ✅ 第一次回复：正常
- ✅ 第二次回复：正常
- ❌ 第三次及以后：不再回复

### 典型场景

```
用户：请帮我生成 5 段内容，每段单独回复
Agent：[第一段内容]
Agent：[第二段内容]
Agent：[沉默...]
```

---

## 问题根因

### 队列系统流程

```
1. 任务入队：enqueueFollowupRun()
   ↓
2. 队列触发：finalizeWithFollowup() → scheduleFollowupDrain()
   ↓
3. 队列执行：循环调用 runFollowupTurn()
   ↓
4. 执行任务：runEmbeddedPiAgent()
```

### 问题所在

**`finalizeWithFollowup` 只在 `agent-runner.ts` 中被调用**

这意味着：
- ✅ 通过 `agent-runner.ts` 启动的 agent 会触发队列
- ❌ 通过其他方式启动的 agent 不会触发队列
- ❌ 队列中的任务执行完成后，不会再次触发队列

### 为什么只能回复 2 次？

1. **第一次回复**：
   - 用户发送消息
   - `agent-runner.ts` 执行 agent
   - `finalizeWithFollowup` 触发队列
   - 队列中有任务，执行第一个任务

2. **第二次回复**：
   - 队列执行第一个任务
   - `followup-runner.ts` 调用 `runEmbeddedPiAgent`
   - ❌ **没有调用 `finalizeWithFollowup`**
   - 队列中还有任务，但没有被触发

3. **第三次及以后**：
   - 队列中的任务永远不会被执行

---

## 解决方案

### 修改文件

`src/auto-reply/reply/followup-runner.ts`

### 修改内容

#### 1. 导入 `finalizeWithFollowup`

```typescript
import { finalizeWithFollowup } from "./agent-runner-helpers.js";
```

#### 2. 在任务完成后触发队列

在 `createFollowupRunner` 函数的最后，`sendFollowupPayloads` 之后添加：

```typescript
await sendFollowupPayloads(finalPayloads, queued);

// 🆕 触发队列继续执行下一个任务
if (queued.run.sessionKey) {
  const queueKey = queued.run.sessionKey;
  finalizeWithFollowup(undefined, queueKey, createFollowupRunner(params));
}
```

### 修复原理

1. **队列系统已存在**：`scheduleFollowupDrain` 已经有递归调用机制
2. **缺少触发**：`followup-runner` 执行完任务后没有触发队列
3. **修复方法**：在任务完成后调用 `finalizeWithFollowup`，触发队列继续执行

### 数据流（修复后）

```
用户消息
  ↓
agent-runner.ts 执行 agent
  ↓
finalizeWithFollowup 触发队列
  ↓
scheduleFollowupDrain 执行第一个任务
  ↓
followup-runner.ts 执行任务
  ↓
🆕 finalizeWithFollowup 触发队列（新增）
  ↓
scheduleFollowupDrain 执行第二个任务
  ↓
followup-runner.ts 执行任务
  ↓
🆕 finalizeWithFollowup 触发队列（新增）
  ↓
... 循环继续，直到队列为空
```

## 关键教训

### 1. 队列系统已存在，不要重复造轮子

**问题**：看到"连续任务只能回复 2 次"，第一反应是"队列系统有问题"

**实际**：队列系统已经完整实现，只是触发机制不完整

**教训**：
- 先理解现有系统
- 不要急于重写
- 找到真正的缺口

### 2. 递归调用需要正确的触发点

**问题**：`scheduleFollowupDrain` 已经有递归调用，为什么队列没有继续执行？

**原因**：递归调用在 `drain.ts` 的 finally 块中，但 `followup-runner` 执行完任务后没有触发队列

**教训**：
- 递归调用需要正确的触发点
- 不能只在一个地方触发
- 需要在每个任务完成后都触发

### 3. 数据流追踪是关键

**方法**：
1. 画出完整的数据流图
2. 找到每个环节的触发点
3. 确认哪个环节缺失

**结果**：
- 发现 `followup-runner` 缺少触发队列的逻辑
- 找到正确的修复位置

### 4. 验证要全面

**不要只看"修复成功"**：
- 要看"不再需要修复"
- 要看"能够连续执行"
- 要看"队列深度变化"

**验证方法**：
- 构建验证
- 代码验证
- 日志验证
- 功能验证

### 5. 最小改动原则

**修复方案对比**：

| 方案 | 改动量 | 风险 | 效果 |
|------|--------|------|------|
| 方案 1：在 followup-runner 中添加触发 | 2 行代码 | 低 | ✅ 完美 |
| 方案 2：在 agent_end Hook 中触发 | 需要重构 | 高 | ❌ 复杂 |
| 方案 3：修改 scheduleFollowupDrain | 需要理解递归逻辑 | 中 | ❌ 不必要 |

**教训**：
- 优先选择最小改动方案
- 不要过度设计
- 简单有效最重要
