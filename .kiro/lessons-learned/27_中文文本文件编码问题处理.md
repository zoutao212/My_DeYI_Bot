# 中文文本文件编码问题处理

> **背景**：读取中文文本文件时出现乱码，通常是编码不匹配导致

---

## 问题现象

### 典型乱码

```

```

### 错误特征

- 中文字符显示为乱码（`��`、`�`）
- 英文和数字正常显示
- 文件内容是中文文本（小说、文档等）
- 使用 UTF-8 读取时出现问题

---

## 根本原因

### 编码不匹配

**问题链条**：
```
文件使用 GBK/GB2312 编码
  ↓
系统/工具默认使用 UTF-8 读取
  ↓
编码不匹配
  ↓
乱码
```

### 常见编码

| 编码 | 使用场景 | 特点 |
|------|----------|------|
| **UTF-8** | 现代系统默认 | 国际标准，支持所有语言 |
| **GBK** | 旧版 Windows 中文系统 | 简体中文，兼容 GB2312 |
| **GB2312** | 早期中文系统 | 简体中文，字符集较小 |
| **Big5** | 繁体中文系统 | 繁体中文 |
| **Shift_JIS** | 日文系统 | 日文 |

### 为什么会出现这个问题？

1. **历史遗留**：
   - 旧版 Windows 中文系统默认使用 GBK
   - 很多旧文件仍然使用 GBK/GB2312
   - 现代系统默认 UTF-8，导致不兼容

2. **文件来源**：
   - 从旧系统复制的文件
   - 从中文网站下载的文件
   - 使用旧版编辑器保存的文件

3. **工具默认**：
   - 现代工具默认 UTF-8
   - 不支持自动编码检测
   - 需要手动指定编码

## 解决方案

### 方案 1：检测编码（第一步）

在处理文件前，先检测正确的编码：

```powershell
# 尝试不同编码读取
$encodings = @("UTF-8", "GBK", "GB2312", "Big5", "Shift_JIS")
$filePath = "文件路径.txt"

foreach ($enc in $encodings) {
    try {
        $content = [System.IO.File]::ReadAllText($filePath, [System.Text.Encoding]::GetEncoding($enc))
        
        # 检查是否有乱码字符
        if ($content -notmatch '�') {
            Write-Host "✅ 可能的编码: $enc"
            Write-Host "前100字符: $($content.Substring(0, [Math]::Min(100, $content.Length)))"
            Write-Host "---"
        }
    } catch {
        # 忽略不支持的编码
    }
}
```

**判断标准**：
- 没有 `�` 字符
- 中文显示正常
- 内容可读

### 方案 2：转换编码为 UTF-8（推荐）

找到正确编码后，转换为 UTF-8：

```powershell
# 1. 使用正确的编码读取
$sourcePath = "原文件.txt"
$targetPath = "原文件_utf8.txt"
$sourceEncoding = "GBK"  # 根据检测结果修改

$content = [System.IO.File]::ReadAllText($sourcePath, [System.Text.Encoding]::GetEncoding($sourceEncoding))

# 2. 保存为 UTF-8（带 BOM）
$utf8WithBom = New-Object System.Text.UTF8Encoding $true
[System.IO.File]::WriteAllText($targetPath, $content, $utf8WithBom)

Write-Host "✅ 已转换为 UTF-8 编码: $targetPath"
```

**注意**：
- UTF-8 有两种：带 BOM 和不带 BOM
- 带 BOM 的 UTF-8 更容易被识别
- 不带 BOM 的 UTF-8 更通用

### 方案 3：使用正确编码读取（临时方案）

如果不想转换文件，每次读取时指定编码：

```powershell
# PowerShell
Get-Content -Path "文件.txt" -Encoding Default  # Windows 默认编码（通常是 GBK）

# 或者明确指定
$content = [System.IO.File]::ReadAllText("文件.txt", [System.Text.Encoding]::GetEncoding("GBK"))
```

**缺点**：
- 每次都要指定编码
- 容易忘记
- 不利于自动化

## 在 Clawdbot 中处理

### 当前问题

Clawdbot 的 `read` 工具默认使用 UTF-8，不支持其他编码。

**影响**：
- 无法直接读取 GBK/GB2312 文件
- 读取时出现乱码
- 需要手动转换

### 临时解决方案

在使用 `read` 工具前，先转换文件编码：

```powershell
# 1. 检测编码
$filePath = "b92e8fa9-0b2b-4e3e-aa19-7b5b9cfe4d47.txt"
$encodings = @("GBK", "GB2312", "UTF-8")

foreach ($enc in $encodings) {
    try {
        $content = [System.IO.File]::ReadAllText($filePath, [System.Text.Encoding]::GetEncoding($enc))
        if ($content -notmatch '�') {
            Write-Host "检测到编码: $enc"
            
            # 2. 转换为 UTF-8
            $utf8Path = $filePath -replace '\.txt$', '_utf8.txt'
            $utf8WithBom = New-Object System.Text.UTF8Encoding $true
            [System.IO.File]::WriteAllText($utf8Path, $content, $utf8WithBom)
            
            Write-Host "已转换: $utf8Path"
            break
        }
    } catch {
        # 继续尝试下一个编码
    }
}

# 3. 使用 read 工具读取 UTF-8 文件
# read path="文件_utf8.txt"
```

### 长期解决方案

修改 `read` 工具以支持编码参数：

**建议修改位置**：`src/agents/pi-tools.read.ts`

**新增参数**：
```typescript
{
  path: string;
  encoding?: "utf-8" | "gbk" | "gb2312" | "big5" | "auto";  // 新增
  // ...
}
```

**实现逻辑**：
1. 如果指定 `encoding`，使用指定编码读取
2. 如果 `encoding="auto"`，尝试检测编码
3. 默认使用 UTF-8

## 最佳实践

### 1. 统一使用 UTF-8

**原则**：所有文本文件统一使用 UTF-8 编码。

**好处**：
- 避免编码问题
- 跨平台兼容
- 工具支持好

**实施**：
- 新建文件时使用 UTF-8
- 旧文件转换为 UTF-8
- 编辑器设置默认 UTF-8

### 2. 文件命名规范

对于编码敏感的文件，在文件名中标注编码：

```
原文件.txt          # 未知编码
原文件_gbk.txt      # GBK 编码
原文件_utf8.txt     # UTF-8 编码
```

### 3. 编码检测脚本

创建一个通用的编码检测和转换脚本：

```powershell
# detect-and-convert.ps1
param(
    [Parameter(Mandatory=$true)]
    [string]$FilePath,
    
    [string]$TargetEncoding = "UTF-8"
)

# 检测编码
$encodings = @("UTF-8", "GBK", "GB2312", "Big5")
$detectedEncoding = $null

foreach ($enc in $encodings) {
    try {
        $content = [System.IO.File]::ReadAllText($FilePath, [System.Text.Encoding]::GetEncoding($enc))
        if ($content -notmatch '�') {
            $detectedEncoding = $enc
            Write-Host "✅ 检测到编码: $enc"
            break
        }
    } catch {
        # 继续尝试
    }
}

if (-not $detectedEncoding) {
    Write-Host "❌ 无法检测编码"
    exit 1
}

# 转换编码
if ($detectedEncoding -ne $TargetEncoding) {
    $content = [System.IO.File]::ReadAllText($FilePath, [System.Text.Encoding]::GetEncoding($detectedEncoding))
    $targetPath = $FilePath -replace '\.txt$', "_$($TargetEncoding.ToLower()).txt"
    
    $utf8WithBom = New-Object System.Text.UTF8Encoding $true
    [System.IO.File]::WriteAllText($targetPath, $content, $utf8WithBom)
    
    Write-Host "✅ 已转换: $targetPath"
} else {
    Write-Host "✅ 文件已是 $TargetEncoding 编码"
}
```

**使用**：
```powershell
.\detect-and-convert.ps1 -FilePath "文件.txt"
```

### 4. 批量转换

如果有多个文件需要转换：

```powershell
# 批量转换当前目录下的所有 .txt 文件
Get-ChildItem -Path . -Filter "*.txt" | ForEach-Object {
    Write-Host "处理: $($_.Name)"
    .\detect-and-convert.ps1 -FilePath $_.FullName
}
```

## 调试检查清单

当遇到文本文件乱码时，按以下步骤检查：

- [ ] **确认文件类型**：是否是文本文件？
- [ ] **检查文件来源**：是否来自旧系统或中文网站？
- [ ] **尝试不同编码**：GBK、GB2312、UTF-8
- [ ] **查看文件头**：是否有 BOM 标记？
- [ ] **转换编码**：转换为 UTF-8
- [ ] **验证结果**：转换后内容是否正常？
- [ ] **更新工具**：是否需要修改 read 工具支持编码参数？

## 常见错误模式

### 错误 1：假设所有文件都是 UTF-8

```powershell
# ❌ 错误：直接使用 UTF-8 读取
Get-Content -Path "文件.txt" -Encoding UTF8

# ✅ 正确：先检测编码
# 使用检测脚本或手动尝试不同编码
```

### 错误 2：转换后不验证

```powershell
# ❌ 错误：转换后不检查
[System.IO.File]::WriteAllText($targetPath, $content, $utf8)

# ✅ 正确：转换后验证
[System.IO.File]::WriteAllText($targetPath, $content, $utf8)
$verify = Get-Content -Path $targetPath -Encoding UTF8 -TotalCount 10
Write-Host "验证前10行: $verify"
```

### 错误 3：混用编码

```powershell
# ❌ 错误：读取时用 GBK，保存时用 UTF-8（但没有转换）
$content = Get-Content -Path "文件.txt" -Encoding Default
Set-Content -Path "文件.txt" -Value $content -Encoding UTF8

# ✅ 正确：明确转换编码
$content = [System.IO.File]::ReadAllText("文件.txt", [System.Text.Encoding]::GetEncoding("GBK"))
[System.IO.File]::WriteAllText("文件_utf8.txt", $content, [System.Text.Encoding]::UTF8)
```

## 相关工具

### Windows

- **记事本**：另存为时可以选择编码
- **Notepad++**：编码菜单 → 转换为 UTF-8
- **VS Code**：右下角显示编码，可以重新打开或保存为其他编码

### Linux/Mac

- **file**：检测文件编码
  ```bash
  file -i 文件.txt
  ```

- **iconv**：转换编码
  ```bash
  iconv -f GBK -t UTF-8 文件.txt > 文件_utf8.txt
  ```

- **enca**：自动检测和转换编码
  ```bash
  enca -L zh_CN -x UTF-8 文件.txt
  ```

## 关键教训

1. **不要假设编码** - 始终检测或明确指定编码
2. **统一使用 UTF-8** - 避免编码问题的最佳方案
3. **转换后验证** - 确保转换正确
4. **工具支持编码参数** - 修改工具以支持不同编码
5. **文档说明编码要求** - 在文档中明确说明编码要求

## 扩展阅读

### 编码基础知识

- **ASCII**：7 位，只支持英文
- **GB2312**：简体中文，双字节
- **GBK**：扩展 GB2312，兼容 GB2312
- **UTF-8**：可变长度，支持所有 Unicode 字符
- **UTF-16**：固定 2 或 4 字节，Windows 内部使用

### BOM（Byte Order Mark）

- **UTF-8 with BOM**：文件开头有 `EF BB BF`
- **UTF-8 without BOM**：没有特殊标记
- **UTF-16 LE**：文件开头有 `FF FE`
- **UTF-16 BE**：文件开头有 `FE FF`

**建议**：
- 文本文件使用 UTF-8 without BOM（更通用）
- 如果需要明确标识，使用 UTF-8 with BOM
