# AI 工具调用歧义问题处理方法论

> **背景**：Telegram 配对崩溃事件 - AI 混淆了节点配对和频道配对两种不同的机制

---

## 问题模式

### 核心问题
AI 看到相似的关键词（如 "pairing"），就错误地认为是同一种操作，导致调用错误的 API/工具。

### 典型表现
1. **关键词触发错误联想**
   - 看到 "pairing code" 就认为是节点配对
   - 实际上有两种完全不同的配对机制

2. **API 调用失败**
   - 调用了错误的 API（`node.pair.approve` 而不是频道配对）
   - 返回 "unknown requestId" 错误

3. **无限循环**
   - AI 不断重试失败的操作
   - 没有识别出致命错误
   - 最终导致系统崩溃

### 根本原因
1. **工具描述不够清晰**
   - 没有明确说明使用场景和限制
   - 没有区分相似但不同的概念

2. **缺少错误提示**
   - AI 无法从错误中学习
   - 错误信息不够友好

3. **文档缺少对比说明**
   - 没有清晰的对比表格
   - 没有明确的使用场景说明

---

## 解决方案

### 第一步：识别歧义

**检查清单**：
- [ ] 是否有多个工具/API 使用相似的关键词？
- [ ] 是否有多个概念容易混淆？
- [ ] 是否有多个使用场景需要区分？

**示例**：
- 节点配对 vs 频道配对
- 文件读取 vs 文件搜索
- 数据库查询 vs 数据库更新

### 第二步：明确工具描述

**原则**：在工具描述中明确说明使用场景和限制

**模板**：
```typescript
description:
  "[工具用途]. " +
  "[使用场景]. " +
  "NOTE: For [其他场景], use [其他工具] instead.",
```

**示例**：
```typescript
description:
  "Discover and control paired nodes (physical devices like phones/tablets). " +
  "Actions: status/describe/pairing/notify/camera/screen/location/run. " +
  "NOTE: For channel user pairing (Telegram/Discord), use exec tool with 'pnpm clawdbot pairing approve <channel> <code>' instead.",
```

### 第三步：添加错误提示

**原则**：在错误处理中添加友好提示，指向正确的工具

**模板**：
```typescript
try {
  // 调用 API
} catch (err) {
  const message = err instanceof Error ? err.message : String(err);
  if (message.includes("[特征错误信息]")) {
    throw new Error(
      `[错误说明]. ` +
      `If you're trying to [其他场景], ` +
      `use: [正确的工具/命令]`
    );
  }
  throw err;
}
```

**示例**：
```typescript
try {
  return jsonResult(
    await callGatewayTool("node.pair.approve", gatewayOpts, { requestId }),
  );
} catch (err) {
  const message = err instanceof Error ? err.message : String(err);
  if (message.includes("unknown requestId")) {
    throw new Error(
      `Unknown node pairing request: ${requestId}. ` +
      `If you're trying to approve a channel user (Telegram/Discord), ` +
      `use: exec({ command: "pnpm clawdbot pairing list <channel>" }) ` +
      `and exec({ command: "pnpm clawdbot pairing approve <channel> <code>" })`
    );
  }
  throw err;
}
```

### 第四步：更新文档

**原则**：在文档中添加对比表格，清晰区分不同的机制

**模板**：
```markdown
## [概念] 说明 ⚠️ 重要

### [类型 1]
- **用途**：[用途说明]
- **工具**：[工具名称]
- **示例**：[使用示例]
- **场景**：[使用场景]

### [类型 2]
- **用途**：[用途说明]
- **工具**：[工具名称]
- **示例**：[使用示例]
- **场景**：[使用场景]

### ⚠️ 关键区别

| 特性 | [类型 1] | [类型 2] |
|------|---------|---------|
| [特性 1] | [值 1] | [值 2] |
| [特性 2] | [值 1] | [值 2] |
| [特性 3] | [值 1] | [值 2] |

**重要**：不要混淆这两种机制！
```

**示例**：见 `clawd/TOOLS.md` 中的"配对机制说明"部分

## 预防措施

### 1. 工具设计原则

**单一职责**：
- 每个工具只做一件事
- 避免功能重叠

**清晰命名**：
- 工具名称要明确表达用途
- 避免使用模糊的名称

**明确限制**：
- 在描述中明确说明不适用的场景
- 提供替代方案

### 2. 错误处理原则

**友好提示**：
- 错误信息要清晰易懂
- 提供具体的解决方案

**指向正确工具**：
- 告诉 AI 应该使用哪个工具
- 提供正确的使用示例

**避免无限循环**：
- 识别致命错误（命令未找到、API 调用错误）
- 添加重试次数限制

### 3. 文档维护原则

**对比说明**：
- 使用表格对比相似概念
- 明确区分使用场景

**示例丰富**：
- 提供正确的使用示例
- 提供错误的使用示例（标注 ❌）

**保持更新**：
- 发现新的歧义时及时更新
- 定期回顾文档的有效性

## 验证清单

修复歧义问题后，必须验证：

- [ ] **工具描述清晰**：明确说明使用场景和限制
- [ ] **错误提示友好**：提供具体的解决方案
- [ ] **文档完整**：有对比表格和使用示例
- [ ] **测试通过**：AI 不再混淆相似概念
- [ ] **无限循环解决**：添加了重试限制和致命错误识别

## 实战案例

### 案例：Telegram 配对崩溃

**问题**：
- AI 看到 "pairing code" 就认为是节点配对
- 调用了错误的 API（`node.pair.approve`）
- 导致无限循环和系统崩溃

**解决**：
1. ✅ 更新 `nodes` 工具描述，明确说明用于物理设备配对
2. ✅ 添加错误提示，指向正确的频道配对命令
3. ✅ 更新 `clawd/TOOLS.md`，添加配对机制对比表格

**效果**：
- AI 不再混淆节点配对和频道配对
- 错误提示清晰，AI 可以自我纠正
- 文档完整，用户可以快速理解

## 关键教训

### 1. 工具描述要清晰
- 不要假设 AI 能自动区分相似概念
- 明确说明使用场景和限制
- 提供替代方案

### 2. 错误提示要友好
- 不要只返回技术错误信息
- 提供具体的解决方案
- 指向正确的工具/命令

### 3. 文档要完整
- 使用对比表格区分相似概念
- 提供丰富的示例
- 保持更新

### 4. 预防胜于治疗
- 设计工具时就要考虑歧义问题
- 定期回顾工具描述和文档
- 发现歧义及时修复
