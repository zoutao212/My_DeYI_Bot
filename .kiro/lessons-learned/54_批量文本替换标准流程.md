# 批量文本替换标准流程

**日期**: 2026-02-02  
**问题类型**: 文本处理、批量操作、代码维护  
**关键词**: 批量替换、PowerShell、正则表达式、文本处理、代码清理

---

## 使用场景

当需要在多个文件中批量替换特定模式时：
- 移除特殊字符（如中文标点符号）
- 统一代码风格
- 批量重命名
- 格式化文本
- 清理冗余内容

---

## 标准流程

### 步骤 1: 搜索匹配的文件

**使用 grepSearch**:
```typescript
grepSearch({
  query: "（.*）",  // 搜索中文括号
  includePattern: "src/**/*.ts",
  explanation: "查找所有包含中文括号的文件"
})
```

**输出**:
- 匹配的文件列表
- 匹配的行号和上下文
- 匹配的数量

### 步骤 2: 分析匹配结果

**检查**:
- 匹配的模式是否正确？
- 是否有误匹配？
- 替换后是否会破坏代码？

**决策**:
- 如果匹配正确 → 继续
- 如果有误匹配 → 调整正则表达式
- 如果会破坏代码 → 手动处理

### 步骤 3: 批量替换

**PowerShell 脚本**:
```powershell
# 单文件替换
$file = "path/to/file.ts"
$content = Get-Content $file -Raw -Encoding UTF8
$content = $content -replace '（[^）]*）', ''  # 移除中文括号及其内容
[System.IO.File]::WriteAllText((Resolve-Path $file).Path, $content, [System.Text.Encoding]::UTF8)
```

**多文件替换**:
```powershell
# 批量替换多个文件
$files = Get-ChildItem -Path "src" -Filter "*.ts" -Recurse
foreach ($file in $files) {
  $content = Get-Content $file.FullName -Raw -Encoding UTF8
  $content = $content -replace '（[^）]*）', ''
  [System.IO.File]::WriteAllText($file.FullName, $content, [System.Text.Encoding]::UTF8)
}
Write-Host "已处理 $($files.Count) 个文件"
```

### 步骤 4: 验证替换结果

**方法 1: 再次搜索**
```powershell
Select-String -Path "src/**/*.ts" -Pattern "（.*）" -Encoding UTF8
```

**预期**: 没有匹配结果

**方法 2: 检查文件内容**
```powershell
Get-Content "path/to/file.ts" -Head 20 -Encoding UTF8
```

**预期**: 中文括号已被移除

### 步骤 5: 构建验证

**命令**:
```powershell
pnpm build
```

**预期**: 构建成功，无错误

**如果失败**:
- 检查替换是否破坏了代码
- 回滚并调整正则表达式
- 手动修复问题文件

## 常用替换模式

### 1. 移除中文括号及其内容

```powershell
$content = $content -replace '（[^）]*）', ''
```

### 2. 替换中文标点符号

```powershell
$content = $content -replace '：', ':'   # 中文冒号 → 英文冒号
$content = $content -replace '；', ';'   # 中文分号 → 英文分号
$content = $content -replace '，', ','   # 中文逗号 → 英文逗号
$content = $content -replace '"', '"'    # 中文引号 → 英文引号
$content = $content -replace '"', '"'    # 中文引号 → 英文引号
```

### 3. 移除多余空格

```powershell
$content = $content -replace '\s+', ' '  # 多个空格 → 单个空格
$content = $content -replace ' \n', '\n' # 行尾空格
```

### 4. 统一换行符

```powershell
$content = $content -replace '\r\n', '\n'  # CRLF → LF
```

### 5. 移除注释

```powershell
$content = $content -replace '//.*$', ''  # 单行注释
$content = $content -replace '/\*[\s\S]*?\*/', ''  # 多行注释
```

## 常见错误

### 错误 1: 编码问题

**问题**: 替换后文件出现乱码

**原因**: 没有指定正确的编码

**解决**:
```powershell
# 始终使用 UTF8 编码
$content = Get-Content $file -Raw -Encoding UTF8
[System.IO.File]::WriteAllText($file, $content, [System.Text.Encoding]::UTF8)
```

### 错误 2: 正则表达式错误

**问题**: 替换了不应该替换的内容

**原因**: 正则表达式过于宽泛

**解决**:
- 使用更精确的正则表达式
- 先在小范围测试
- 使用 `-match` 预览匹配结果

### 错误 3: 破坏代码结构

**问题**: 替换后代码无法编译

**原因**: 替换了代码中的关键字符

**解决**:
- 使用更精确的正则表达式
- 排除特定文件或目录
- 手动处理特殊情况

## 实战案例

### 案例 1: 移除中文括号

**背景**: 系统提示词中的中文括号干扰 LLM 解析

**步骤**:
1. 搜索: `grepSearch -query "（.*）" -includePattern "src/agents/system-prompt*.ts"`
2. 替换: `$content = $content -replace '（[^）]*）', ''`
3. 验证: `pnpm build`
4. 测试: 重启 Gateway 并测试工具调用

**结果**: 
- ✅ 移除了所有中文括号
- ✅ 系统提示词大小减少 75%
- ✅ LLM 工具调用恢复正常

### 案例 2: 统一代码风格

**背景**: 代码中混用了单引号和双引号

**步骤**:
1. 搜索: `grepSearch -query "'" -includePattern "src/**/*.ts"`
2. 替换: `$content = $content -replace "'", '"'`
3. 验证: `pnpm lint`
4. 提交: `git commit -m "style: unify quotes"`

**结果**:
- ✅ 统一使用双引号
- ✅ 通过 lint 检查

## 自动化脚本

### 通用替换脚本

```powershell
# replace-pattern.ps1
param(
  [string]$Pattern,
  [string]$Replacement,
  [string]$Path,
  [string]$Filter = "*.ts"
)

$files = Get-ChildItem -Path $Path -Filter $Filter -Recurse
$count = 0

foreach ($file in $files) {
  $content = Get-Content $file.FullName -Raw -Encoding UTF8
  $newContent = $content -replace $Pattern, $Replacement
  
  if ($content -ne $newContent) {
    [System.IO.File]::WriteAllText($file.FullName, $newContent, [System.Text.Encoding]::UTF8)
    $count++
    Write-Host "✓ $($file.Name)"
  }
}

Write-Host "`n已处理 $count 个文件"
```

**使用**:
```powershell
.\replace-pattern.ps1 -Pattern "（[^）]*）" -Replacement "" -Path "src" -Filter "*.ts"
```

## 关键教训

1. **先搜索，再替换**
   - 使用 grepSearch 确认匹配结果
   - 避免盲目替换

2. **使用正确的编码**
   - 始终使用 UTF8
   - 避免乱码问题

3. **分批处理**
   - 先测试一个文件
   - 验证成功后再批量处理

4. **备份和版本控制**
   - 使用 Git 跟踪修改
   - 可以随时回滚

5. **验证替换结果**
   - 再次搜索确认
   - 构建验证
   - 测试功能
