# AI 工具使用陷阱 ⚠️ 极其重要

## 工具调用成功 ≠ 文件实际写入 ⚠️ 血泪教训

### 问题描述

- **问题**：`strReplace`、`fsWrite` 等工具调用显示成功，但文件实际上没有写入磁盘
- **表现**：
  - 工具返回 "Replaced text in xxx" 或 "File written"
  - 但用 PowerShell `Get-Content` 读取文件，内容没变
  - 用户刷新浏览器，看不到任何变化

### 根本原因

- 工具可能有内部缓存
- 写入操作被某种机制阻止
- 工具返回值只代表"请求成功"，不代表"操作成功"

### 解决方案

#### 1. 永远用 PowerShell 验证

修改后立即用 `Get-Content` 或 `Select-String` 验证：

```powershell
# 验证文件内容
Get-Content "path/to/file" -Head 10 -Encoding UTF8

# 搜索特定内容
Select-String -Path "path/to/file" -Pattern "关键词" -Encoding UTF8

# 检查文件大小和修改时间
Get-Item "path/to/file" | Select Name, Length, LastWriteTime
```

#### 2. 不要相信工具的返回值

必须独立验证，不要只看工具的返回信息。

#### 3. 如果工具失败，改用 PowerShell 直接写入

```powershell
# 读取文件
$content = Get-Content $filePath -Raw -Encoding UTF8

# 替换内容
$newContent = $content -replace 'old', 'new'

# 写回文件
[System.IO.File]::WriteAllText($filePath, $newContent, [System.Text.Encoding]::UTF8)
```

### 教训

**工具调用成功只是"请求成功"，不代表"操作成功"。**

## 文件读取工具的缓存问题

### 问题描述

- **问题**：`readFile` 工具可能返回缓存的旧内容，而不是磁盘上的最新内容
- **表现**：工具显示文件有某段代码，但 PowerShell 读取显示没有

### 解决方案

1. **关键验证必须用 PowerShell**：不要只依赖 readFile
2. **如果怀疑缓存**：用 `Get-Content -Head 10` 快速验证

### 教训

**调试时不要只依赖一种工具，要交叉验证。**

## 修改大文件的正确姿势

### 问题描述

- **问题**：大文件（>1000行）的修改容易出问题

### 解决方案

1. **先用 `Select-String` 找到要修改的行号**

```powershell
Select-String -Path $file -Pattern "目标文本" | Select LineNumber
```

2. **用 `Get-Content` 读取周围几行确认上下文**

```powershell
$lines = Get-Content $file -Encoding UTF8
$lines[550..560]
```

3. **用 PowerShell 的字符串替换或行插入来修改**

```powershell
# 修改并写回
$lines[556] = "新内容"
[System.IO.File]::WriteAllLines($file, $lines, [System.Text.Encoding]::UTF8)
```

4. **修改后立即验证**

```powershell
Get-Content $file -Encoding UTF8 | Select-Object -Index 556
```

## 文件修改验证规范 ⚠️ 极其重要

### 触发场景

每次修改文件后。

### 标准流程

1. **用 PowerShell 验证文件内容**

```powershell
Get-Content "path/to/file" -Head 10 -Encoding UTF8
```

2. **搜索关键代码片段**

```powershell
Select-String -Path "path/to/file" -Pattern "关键代码" -Encoding UTF8
```

3. **添加可观察标记**

修改时加入版本号或标题变化，便于快速确认：

```html
<!-- VERSION: 2026-01-02-v3 -->
<title>xxx v3</title>
```

### 原因

**工具调用显示成功不代表文件真的被修改，必须独立验证！**

- ✅ 不相信工具的返回值
- ✅ 添加可观察标记
